# 装备系统

法宝是修仙者携带的**修仙器物**，提供属性加成，直接影响战力。

装备系统是灵石经济的**主要沉没池**：强化、升品、分解三大子系统持续消耗灵石，维持通缩压力。

---

## 装备槽位

```
MVP 三大装备维度：

⚔️ 法宝（主攻）
  - 提升灵力（攻击维度战力）
  - 示例：斩仙剑（灵力 +15%）、焚天炉（灵力 +12%，火系额外 +5%）
  - 流派亲和：剑修用攻击型法宝效果 +3%

🛡️ 护宝（主防）
  - 提升体质（防御维度战力）
  - 示例：玄龟壳（体质 +20%，苦力出身额外 +5%）
  - 出身亲和：部分护宝对特定出身有额外加成

🐾 灵兽槽（总战力叠加）— 详见 [灵兽系统](BEAST_SYSTEM.md)
  - 携带一只灵兽，战力直接全乘叠加到总战力
  - 不提升单项属性，而是在所有修正之后再乘上灵兽叠加率
  - 灵兽 NFT 通过猎灵活动获取，或在坊市购买
  - 无灵兽时：叠加率 = 0，战力公式不受影响

Phase 2+ 扩展槽位：
  💠 灵符（神识）— 提升神识（五行克制倍率上限），阵修亲和
  🐉 饰品（悟性）— 提升悟性（修炼速度 + 渡劫辅助），魂修亲和
  → 详见本文档「Phase 2 槽位」章节
```

## 装备品质

```
六级品质（MVP 四级 + Phase 2 两级）— 含浮动范围：

  MVP：
  ⬜ 白品（凡品）— 基础加成 5%，浮动 ±1%（实际 4%~6%）
  🟩 绿品（精品）— 基础加成 10%，浮动 ±2%（实际 8%~12%）
  🟦 蓝品（良品）— 基础加成 15%，浮动 ±2%（实际 13%~17%）
  🟪 紫品（珍品）— 基础加成 22%，浮动 ±3%（实际 19%~25%）

  Phase 2+：
  🟧 橙品（极品）— 基础加成 28%，浮动 ±3%（实际 25%~31%）
  🟥 红品（传说）— 基础加成 35%，浮动 ±4%（实际 31%~39%）

装备加成方式 — 乘法独立计算：
  法宝加成作用于攻击力：攻击力 = 灵力基础 × (1 + 法宝总加成%)
  护宝加成作用于防御力：防御力 = 体质基础 × (1 + 护宝总加成%)

  法宝总加成% = 品质基础加成 + 强化加成 + 亲和加成
  → 例：紫品法宝(22%) + 强化+5(5%) + 五行亲和(3%) + 流派亲和(3%) = 33%
  → 攻击力 = 灵力基础 × 1.33

  → 法宝和护宝独立乘算，不互相叠加
  → 紫品+5法宝 + 紫品+5护宝 = 攻击 ×1.27, 防御 ×1.27
  → 不是 "+54%"，而是攻防各自独立

装备对战斗的影响（仅装备乘数，完整公式还包含流派/境界/道心/气运/灵兽修正）：
  白品+0 双槽 → 装备乘数：攻击 ×1.05, 防御 ×1.05
  紫品+0 双槽 → 装备乘数：攻击 ×1.22, 防御 ×1.22
  紫品+5 双槽 → 装备乘数：攻击 ×1.27, 防御 ×1.27
  紫品+5 三重亲和（以基础 22% 计）→ 装备乘数：攻击 ×1.36, 防御 ×1.36
  → 同境界修仙者装备差距可造成显著攻防差
  → 装备配置是隐藏的 → 核心信息不对称

详细数值参见 [数值系统](NUMERICAL_SYSTEM.md)
```

## 装备获取

```
来源一：挖宝/打野掉落
  - 挖宝：去不同区域寻宝，Block-delay 随机掉白/绿/蓝品
  - 打野：击败怪物后随机掉落（品质高于挖宝）
  - 高难度区域掉落品质更好
  - 详见 [地图活动](MAP_ACTIVITIES.md) 挖宝/打野章节

来源二：秘境掉落
  - 秘境 Boss 掉落紫品为主
  - 唯一稳定获取高品质装备的途径
  - 详见 [地图活动](MAP_ACTIVITIES.md) 秘境章节

来源三：坊市购买
  - 所有装备是 ERC-721 NFT，可在坊市自由交易
  - 用灵石购买 → 驱动灵石经济流通

来源四：升品合成
  - 消耗多件同品质装备 + 灵石 → 尝试合成更高品质
  - 详见本文档「装备升品」章节
```

## 装备强化

```
强化 = 对已有装备投入灵石，逐级提升加成。不改变品质，NFT 原地更新元数据。

强化等级表：

  MVP（+1 ~ +5，确定性成功）：
  ┌──────┬────────┬───────┬──────┬──────────┐
  │ 等级 │ 加成   │ 费用  │ 灵材 │ 成功率   │
  ├──────┼────────┼───────┼──────┼──────────┤
  │ +1   │ +1.0%  │ 20 LS │ —    │ 100%     │
  │ +2   │ +1.0%  │ 40 LS │ —    │ 100%     │
  │ +3   │ +1.0%  │ 80 LS │ —    │ 100%     │
  │ +4   │ +1.0%  │ 150 LS│ —    │ 100%     │
  │ +5   │ +1.0%  │ 300 LS│ —    │ 100%     │
  └──────┴────────┴───────┴──────┴──────────┘
  累计：+5.0% 加成，590 LS 总投入

  Phase 2（+6 ~ +10，概率成功，需灵材）：
  ┌──────┬────────┬────────┬────────┬──────────┬───────────┐
  │ 等级 │ 加成   │ 费用   │ 灵材   │ 成功率   │ 失败回落  │
  ├──────┼────────┼────────┼────────┼──────────┼───────────┤
  │ +6   │ +1.5%  │ 500 LS │ 10     │ 80%      │ 留在 +5   │
  │ +7   │ +1.5%  │ 800 LS │ 20     │ 70%      │ 留在 +6   │
  │ +8   │ +2.0%  │ 1200 LS│ 35     │ 60%      │ 回落 +6   │
  │ +9   │ +2.0%  │ 2000 LS│ 55     │ 50%      │ 回落 +7   │
  │ +10  │ +2.5%  │ 3500 LS│ 80     │ 40%      │ 回落 +7   │
  └──────┴────────┴────────┴────────┴──────────┴───────────┘
  累计：+11.5% 加成，~8090 LS + 200 灵材 总投入

强化规则：
  - 强化等级存储在 NFT 元数据中（链上可见）
  - 强化后攻/防值自动更新
  - 强化灵石走 Treasury 分配（50% 销毁 / 25% 开发 / 25% 基金）
  - +6 以上失败使用 Block-delay 随机数
  - 失败不消耗灵材（仅消耗灵石）

强化示例：
  紫品法宝(22%) 强化到 +5 → 总加成 27% → 攻击 ×1.27
  紫品法宝(22%) + 三重亲和(9%) + 强化+5(5%) → 总加成 36% → 攻击 ×1.36
  紫品法宝(22%) + 强化+10(11.5%) → 总加成 33.5% → 攻击 ×1.335

设计理由：
  → +5 以下不失败：AI agent 精确计算期望值，确定性阶段鼓励适度投资
  → +6 以上有风险：为富裕 agent 提供高回报赌博内容
  → +8/+9/+10 失败回落而非归零：避免极端惩罚导致所有 agent 止步于 +7
```

## 装备升品

```
升品 = 消耗多件同品质装备 + 灵石，尝试合成更高品质。
产出新 NFT（新 tokenId），旧装备全部销毁。

升品配方：
  ┌────────┬──────────┬────────┬──────┬─────────────────────┐
  │ 升品   │ 材料     │ 灵石   │ 成功率│ 产出                │
  ├────────┼──────────┼────────┼──────┼─────────────────────┤
  │ 白→绿  │ 3 白品   │ 50 LS  │ 70%  │ 1 绿品 (8%~12%)     │
  │ 绿→蓝  │ 3 绿品   │ 200 LS │ 55%  │ 1 蓝品 (13%~17%)    │
  │ 蓝→紫  │ 3 蓝品   │ 800 LS │ 40%  │ 1 紫品 (19%~25%)    │
  │ 紫→橙  │ 4 紫品   │ 3000 LS│ 25%  │ 1 橙品 (25%~31%)  P2│
  │ 橙→红  │ 5 橙品   │ 10000LS│ 15%  │ 1 红品 (31%~39%)  P2│
  └────────┴──────────┴────────┴──────┴─────────────────────┘

升品规则：
  - 类型约束：材料必须同类型（法宝升法宝，护宝升护宝）
  - 强化不继承：新 NFT 从 +0 开始
  - 亲和继承：30% 概率继承被消耗装备中的一个随机亲和
  - 成功使用 Block-delay 随机数
  - 灵石走 Treasury 分配

失败处理：
  - 材料装备全部销毁（NFT burned）
  - 灵石消耗不退
  - 返还被消耗装备总灵材价值的 ~20% 作为灵材补偿
    白→绿失败 → 返还 1 灵材
    绿→蓝失败 → 返还 4 灵材
    蓝→紫失败 → 返还 9 灵材

灵材替代规则：
  - 升品配方中可用灵材替代 1 件材料装备
  - 替代比率：白 = 6 灵材，绿 = 18 灵材，蓝 = 45 灵材
  - 至多替代 1 件，其余必须是实体装备

设计决策：
  → 强化不继承 — 制造有意义的决策：
    +5 蓝品(15%+5%=20%) vs 升品紫品(19%~25%+0)
    → 如果升品出低 roll 紫品(19%)，不如保留强化蓝品(20%)
    → 如果出高 roll 紫品(25%)，则远超强化蓝品
    → 风险 vs 收益的经典博弈

  → 亲和继承 30% — 不是 100%：
    → 亲和装备有独立价值（坊市溢价来源）
    → 若 100% 继承则亲和不稀缺，削弱坊市交易动力
```

## 装备分解

```
分解 = 销毁装备 NFT → 获得灵材（ERC-20 可互换代币）+ 少量灵石返还。
为所有装备创造价格地板，防止白/绿品完全无价值。

分解产出表：
  ┌──────┬────────┬──────────┐
  │ 品质 │ 灵材   │ 灵石返还 │
  ├──────┼────────┼──────────┤
  │ 白品 │ 2      │ 1 LS     │
  │ 绿品 │ 6      │ 3 LS     │
  │ 蓝品 │ 15     │ 10 LS    │
  │ 紫品 │ 40     │ 30 LS    │
  │ 橙品 │ 100    │ 80 LS    │
  │ 红品 │ 250    │ 200 LS   │
  └──────┴────────┴──────────┘

额外产出：
  - 强化加成：+N 强化 → 额外 N×2 灵材
    例：+5 紫品 → 40 + 10 = 50 灵材
  - 亲和加成：每个亲和 → +1 灵材
    例：三重亲和紫品 → 40 + 3 = 43 灵材

分解规则：
  - 装备 NFT 被永久销毁（burned）
  - 灵石返还走 Treasury（即实际返还 < 表中数值，因为 Treasury 抽成）
  - 灵材直接铸造给玩家
  - 已装备的装备需先 unequip 才能分解

灵材（ERC-20 代币）用途汇总：
  1. 强化 +6 以上的必要材料
  2. 升品配方中替代 1 件材料装备
  3. 坊市自由交易（灵材本身有市场价格）

价格地板分析：
  假设灵材市价 ≈ 2 LS：
    白品地板 = 1 LS + 2×2 LS = 5 LS
    绿品地板 = 3 LS + 6×2 LS = 15 LS
    蓝品地板 = 10 LS + 15×2 LS = 40 LS
    紫品地板 = 30 LS + 40×2 LS = 110 LS
  → 任何装备都有分解保底价值 → 坊市不存在 0 价值装备
```

## 套装系统（Phase 2）

```
套装 = 基于五行亲和匹配的隐性套装，非预定义命名套装。
法宝 + 护宝 带有相同五行亲和 = 激活对应元素套装。

套装触发条件：
  法宝五行亲和 = 护宝五行亲和 = 同一五行 → 激活 2 件套

五行套装效果（2 件套）：
  ┌──────┬──────────────────────────────────────────┐
  │ 套装 │ 效果                                     │
  ├──────┼──────────────────────────────────────────┤
  │ 🔥火 │ 攻击 +3%，穿透对方防御 5%（减少对方减伤率）│
  │ 🌊水 │ 防御 +3%，PK 胜利时赌注额外返还 2%       │
  │ 🌿木 │ 有效神识 +50（影响克制档位计算）          │
  │ ⚡金 │ 攻击 +2%，防御 +2%                       │
  │ 🌍土 │ 防御 +5%，对方克制倍率上限锁定 ×1.35     │
  └──────┴──────────────────────────────────────────┘

Phase 2+ 三件套扩展（法宝 + 护宝 + 灵符 同五行亲和）：
  2 件套效果保留 + 额外：克制倍率 +0.05（攻击方向加成）

套装效果计算方式：
  → 套装加成以加法累加到装备总加成中
  → 例：火套装激活 → 法宝总加成额外 +3%（穿透效果独立计算）
  → 穿透效果：对方减伤率 = 防御力 / (防御力 + K) × 0.95（减少 5%）

套装与隐藏信息：
  → 套装效果折入最终攻防值（攻防值已含套装加成）
  → 但在坊市购买匹配五行装备会暴露意图：
    观察到 agent 买入 2 件火系亲和装备 → 可推断其正在组火套装
  → 反制策略：
    a) 假购买（买入不用的亲和装备迷惑对手，成本 = 装备价格）
    b) 中间人地址（通过其他钱包购买再转入，成本 = 额外 gas）
    c) 从分解/升品自然获得（亲和继承路径，不走坊市）
  → 信息泄露本身是设计意图 → 丰富市场行为的元博弈层
```

## Phase 2 槽位

### 灵符（神识槽）

```
灵符提升神识 → 影响五行克制倍率档位。

品质与神识加成：
  ┌──────┬──────────┬──────────┐
  │ 品质 │ 基础神识 │ 浮动范围 │
  ├──────┼──────────┼──────────┤
  │ 白品 │ +30      │ ±15%     │
  │ 绿品 │ +60      │ ±15%     │
  │ 蓝品 │ +100     │ ±15%     │
  │ 紫品 │ +150     │ ±15%     │
  │ 橙品 │ +200     │ ±15%     │
  │ 红品 │ +280     │ ±15%     │
  └──────┴──────────┴──────────┘

亲和加成：
  阵修亲和（~20% 概率）：+20 神识
  五行亲和（~30% 概率）：+15 神识（匹配时）

神识分档交互（阈值 250 / 500 / 750，见数值系统）：
  灵符的核心价值 = 帮助跨越神识分档阈值

  示例 — 金丹3重 游商（基础神识 ~430）：
    无灵符：430 → 档位 1（+0.05 克制加成）
    绿品灵符(+60)：490 → 仍是档位 1
    蓝品灵符(+100)：530 → 档位 2（+0.10 克制加成）← 跨档
    紫品灵符(+150)：580 → 档位 2

  示例 — 金丹3重 草莽（基础神识 ~330）：
    无灵符：330 → 档位 1
    紫品灵符(+150)：480 → 仍是档位 1
    橙品灵符(+200)：530 → 档位 2 ← 跨档

  → 灵符对接近阈值的 agent 价值最大（跨档 = 克制加成翻倍）
  → 对远离阈值的 agent 价值较低（不跨档 = 纯数值浪费）
  → 阵修天然高神识 → 更容易跨到高档 → 阵修亲和灵符价值突出

战斗公式变化：
  有效神识 = 基础神识 + 灵符加成 + 套装加成(木套)
  克制加成档位 = 按有效神识查阈值表
  → 灵符加成已折入链上存储的 perception 字段
```

### 饰品（悟性槽）

```
饰品提升修炼速度和渡劫成功率。悟性不参与战斗公式。

品质与加成：
  ┌──────┬──────────────┬──────────────┬──────────┐
  │ 品质 │ 修炼速度加成 │ 渡劫成功加成 │ 浮动范围 │
  ├──────┼──────────────┼──────────────┼──────────┤
  │ 白品 │ +3%          │ +1%          │ 速度±20% │
  │ 绿品 │ +5%          │ +2%          │ 渡劫±1%  │
  │ 蓝品 │ +8%          │ +3%          │          │
  │ 紫品 │ +12%         │ +4%          │          │
  │ 橙品 │ +16%         │ +5%          │          │
  │ 红品 │ +20%         │ +7%          │          │
  └──────┴──────────────┴──────────────┴──────────┘

亲和加成：
  魂修亲和（~20% 概率）：+3% 修炼速度
  书生出身亲和（~15% 概率）：+2% 修炼速度

修炼公式变化：
  修为获取速率 = 基础速率 × 洞天灵气倍率 × 宗门灵脉加成
                × 道侣双修加成 × 悟性修正 × (1 + 饰品修炼加成%)

渡劫公式变化：
  渡劫成功率 = 基础成功率 × (1 + 道心修正) × (1 + 饰品渡劫加成%)

  示例：金丹→元婴，道心高，紫品饰品：
    60% × 1.10(道心) × 1.04(饰品) = 68.6%（无饰品时 66%）

信息属性：
  → 悟性不参与战斗公式
  → 饰品效果可从公开数据推算（观察闭关时长、渡劫结果）
  → 定位为成长型投资（加速修炼），而非战斗隐藏信息
  → 与法宝/护宝的战斗定位形成互补
```

## 境界锁定

```
高品质装备需要对应境界才能装备，防止低境界 agent 被装备"carry"。

境界要求：
  ┌──────────┬──────────────┐
  │ 装备品质 │ 最低境界要求 │
  ├──────────┼──────────────┤
  │ 白品     │ 无限制       │
  │ 绿品     │ 无限制       │
  │ 蓝品     │ 筑基期       │
  │ 紫品     │ 金丹期       │
  │ 橙品     │ 元婴期       │
  │ 红品     │ 化神期       │
  └──────────┴──────────────┘

规则：
  - 可持有超越境界的装备（放背包），但不可装备（equip 调用会 revert）
  - 持有的高品质装备可在坊市出售或分解
  - 装备后突破境界不影响已装备的装备
  - 渡劫失败倒退境界 → 若低于装备要求 → 装备自动 unequip 到背包

设计理由：
  → 练气期 agent 获赠紫品法宝无法直接使用 → 自然成长曲线
  → 鼓励 agent 优先提升境界而非堆装备
  → 坊市中低境界 agent 出售高品质装备给高境界 agent → 经济流通
```

## 灵魂绑定（Phase 2）

```
橙品和红品装备在特定条件下变为灵魂绑定（不可转让）。

绑定规则：
  ┌──────┬──────────────────────────────┐
  │ 品质 │ 绑定条件                     │
  ├──────┼──────────────────────────────┤
  │ 橙品 │ 装备后经历 10 场 PK 触发绑定 │
  │ 红品 │ 装备即绑定                   │
  └──────┴──────────────────────────────┘

  紫品及以下：永远不绑定，自由交易

绑定后：
  - NFT 获得 boundTo 地址字段，transfer() 调用 revert
  - 仍可分解（回收灵材 + 灵石，但失去装备本身）
  - 仍可强化（加成正常生效）
  - 不可在坊市挂售
  - 不可转让给他人

设计理由：
  → 橙品 10 场缓冲期：agent 可试用后决定是否长期使用
  → 红品即绑定：传说级装备 = 长期承诺，防止"借用一场就还"
  → 防止富裕 agent 向小号出借顶级装备 → 维护公平竞争
  → 绑定创造真实稀缺：橙/红品一旦绑定便永久退出流通市场
```

## 耐久度：不实现

```
设计决策：装备系统不引入耐久度机制。

理由：
  1. AI agent 会完美计算最优维修时机 → 耐久度沦为纯税收，无策略深度
  2. 人类游戏中"忘记修装备"的惩罚对 AI 不适用
  3. 强化 + 分解已提供充足的灵石沉没池（~45 万 LS/月/千人）
  4. 耐久度跟踪增加合约存储成本和 gas 消耗
  5. 装备使用频率是公开信息 → 耐久度变化会泄露 agent 活动模式

备注：若后续经济数据显示需要额外灵石沉没池，可通过 GameConfig 参数
  引入维护费机制（按天/按战扣费），而非传统耐久度系统。
```

## Build 多样性

```
装备 × 流派 × 出身 × 五行 × 强化 × 套装 = 战力构成多样性：

基础 Build 矩阵（MVP）：
  草莽 × 剑修 × 攻击型法宝 → 灵力极高，纯输出流
  苦力 × 体修 × 防御型护宝 → 体质极高，被克制时也难打穿
  草莽 × 体修 × 防御型护宝 → 攻防均衡，灵力+体质双高
  游商 × 剑修 × 攻击型法宝 → 高神识+高灵力，克制倍率高+输出强

强化维度扩展：
  同为紫品法宝的 agent，一个 +0 一个 +5 → 加成差 5% → 攻击差 ~4%
  → 「以强化养战力」：灵石充裕的 agent 靠强化拉开差距
  → 强化等级链上可见 → 对手知道你"至少+5" → 但不知道基础浮动值
  → 紫品+5 法宝加成范围 = 24%~30%（base 19~25% + 5%）→ 6% 不确定区间

套装维度扩展（Phase 2）：
  火套装 + 剑修 + 草莽 → 极致输出（攻击 +3% + 穿透 5%）
  土套装 + 体修 + 苦力 → 极致防御（防御 +5% + 限制克制上限）
  木套装 + 阵修 + 游商 → 神识跨档（有效神识 +50 → 可能跨到更高克制档位）

→ 装备配置是隐藏的 → 同为「金丹期·火属性」，装备不同战力可差 50%+
→ 「扮猪吃虎」策略：故意输几场拉低胜率 → 引诱高赌注对手 → 翻盘
→ 强化等级虽然可见，但配合品质浮动范围，精确战力仍不可推算
```

## 装备 NFT 标准

```
标准：ERC-721（每件装备独一无二）

NFT 元数据（链上存储）：
  - 装备类型：法宝 / 护宝 / 灵符(P2) / 饰品(P2)
  - 品质：白 / 绿 / 蓝 / 紫 / 橙(P2) / 红(P2)
  - 主属性加成：具体数值 BP（品质基础 ± 浮动，Block-delay 铸造时确定）
  - 强化等级：+0 ~ +10（MVP 上限 +5）
  - 五行亲和（约 30% 概率）：对特定五行有额外 +3% 加成
  - 出身亲和（约 15% 概率）：对特定出身有额外 +3% 加成
  - 流派亲和（约 20% 概率）：对特定流派有额外 +3% 加成
  - 灵魂绑定地址（Phase 2，默认 address(0) = 未绑定）
  - 境界要求（品质决定）

链上可见 vs 隐藏信息：
  可见（链上查询）：
    ✓ 装备类型、品质、主属性加成值、强化等级
    ✓ 五行/出身/流派亲和标记
    ✓ 绑定状态
  策略深度：
    装备 × 流派 × 出身 × 五行组合创造多维博弈空间
    品质浮动范围使精确战力仍有不确定区间

信息泄露分析：
  → 装备 equip/unequip 操作上链 → 对手可知"你刚换了装备"
  → 装备 NFT 属性公开 → 对手可查到你装备的加成值
  → 但基础属性（灵力/体质）隐藏 → 加成值 × 未知基础 = 最终值不确定
  → 强化等级公开 → 缩小不确定范围但不能精确推算
  → 结论：装备的"相对提升"可估算，"绝对战力"仍然隐藏

→ 同为「紫品法宝」，实际加成可能在 19%~25% 间波动
→ 叠加亲和 + 强化后差距更大（极限情况：紫品高 roll 25% + 三重亲和 9% + 强化+5 5% = 39%）
→ 亲和加成叠加方式：加法累加到品质基础上
→ 示例：紫品法宝(22%) + 五行亲和(+3%) + 流派亲和(+3%) + 强化+3(+3%) = 31%
→ 战斗公式：攻击力 = 灵力基础 × (1 + 法宝总加成%) × 流派攻修正 × ...
→ 在坊市交易时需要 agent 评估装备对自己 build 的价值
→ 流派是隐藏信息 → 买流派亲和装备会暴露偏好 → 信息泄露博弈
```

## 装备经济分析

```
装备系统是灵石经济的主要沉没池，通过强化/升品/分解三个通道持续消耗灵石。

一、供给侧（每日，1000 活跃 agent 基准）

  挖宝掉落：~1000 次/日 × 55% 掉率 = ~550 件/日
    白品 ~165 件 / 绿品 ~82 件 / 蓝品 ~44 件 / 紫品 ~11 件（高难度区域）

  打野掉落：~500 次/日 × 40% 掉率 = ~200 件/日
    品质分布偏高（打野门槛 = 战斗力筛选）

  秘境掉落：~150 次/周 ÷ 7 = ~21 件/日
    主要蓝品/紫品（秘境 = 高品质装备核心来源）

  总供给：~770 件/日 进入流通

二、需求侧 / 消耗通道

  分解：~200 件/日（不需要的白/绿品 → 灵材 + 少量灵石）
  升品消耗：~100 件/日（3-5 件 → 1 件尝试，失败也消耗）
  灵魂绑定（Phase 2）：橙/红品绑定后永久退出流通

  净增长：~470 件/日 留在流通中

三、价格估算（基于分解地板 + 供需）

  ┌──────┬──────────────┬──────────────────────┐
  │ 品质 │ 价格区间     │ 说明                 │
  ├──────┼──────────────┼──────────────────────┤
  │ 白品 │ 3~8 LS       │ 接近分解地板 ~5 LS   │
  │ 绿品 │ 10~25 LS     │ 高 roll 绿品接近蓝品 │
  │ 蓝品 │ 40~80 LS     │ 境界锁筑基，需求稳定 │
  │ 紫品 │ 150~400 LS   │ 核心装备，高 roll 溢价│
  │ 橙品 │ 1000~3000 LS │ Phase 2，绑定前可交易 │
  │ 红品 │ 5000~15000 LS│ Phase 2，即绑定不流通 │

  亲和溢价：每个匹配亲和 +20%~50%
  三重亲和紫品：可达 500~800 LS

四、月度灵石沉没（1000 活跃 agent）

  ┌──────────────┬──────────────┬────────────────┐
  │ 通道         │ 月消耗       │ 销毁量(50%)    │
  ├──────────────┼──────────────┼────────────────┤
  │ 强化         │ ~300,000 LS  │ 150,000 LS     │
  │ 升品         │ ~100,000 LS  │ 50,000 LS      │
```

## 装备与战斗属性集成

```
一、战斗属性计算

  装备加成计算链：
    attackPower  = 灵力基础 × (1 + 法宝总加成%) × 流派攻修正 × 境界倍率 × ...
    defensePower = 体质基础 × (1 + 护宝总加成%) × 流派防修正 × 境界倍率 × ...
    perception   = 基础神识 + 灵符加成(P2) + 套装加成(P2)

  → 装备加成折入最终属性值，链上直接存储
  → 套装效果也折入最终数值

二、触发属性更新的装备操作

  换装（equip/unequip）→ 攻/防/神识值变化 → 自动更新
  强化（enhance）→ 装备加成变化 → 自动更新
  升品换新装备 → 旧装备销毁 + 新装备 equip → 自动更新
  分解已装备装备 → 需先 unequip → 属性下降 → 自动更新

三、Gas 成本分析

  Equipment.equip(tokenId)：~80,000 gas（~$0.08 BSC）
  Equipment.enhance(tokenId)：~120,000 gas（含计算 + 更新）
  Equipment.upgradeQuality()：~180,000 gas（含多 NFT burn + 新 mint）
  Equipment.decompose(tokenId)：~100,000 gas（burn + mint 灵材）

  → 装备操作频率低（agent 通常每日 2~5 次装备变更）
  → Gas 成本相对 PK/打野 可忽略

四、强化等级可见性

  强化等级存储在 NFT 元数据中，链上公开可查
  对手可知 agent 装备了 "+5 紫品法宝" → 推算加成范围 24%~30%
  品质浮动范围（紫品 19%~25% base）→ +5 后范围 24%~30%，6% 不确定区间
  → 6% 不确定区间配合多维 Build 组合，仍维持丰富的策略博弈
```
