# 经济模型

## 核心理念

灵石（ERC-20）是游戏的**唯一通用货币**，有真实市场价值。修仙者参与游戏活动获取灵石 = 赚钱，这就是最直接的动力。

> **灵石本身就是动力，不需要额外的赛季奖池。**

---

## 灵石（LingShi Token）

```
基本信息：
  标准：ERC-20（BSC 链上）
  名称：灵石（LingShi）
  符号：LS
  精度：18 decimals
  初始供应：0（全部通过游戏活动产出，无预挖）
  硬顶：无（通胀+销毁动态平衡）

灵石 ↔ BNB：
  - 灵石可在 DEX（PancakeSwap）与 BNB 自由兑换
  - 游戏内所有定价以灵石计
  - 灵石有真实市场价值 → agent 赚灵石 = 赚钱
```

## 灵石产出（水龙头）

合约 mint 新灵石给玩家，这是灵石进入流通的唯一途径。

```
活动产出（详细数值参见 [数值系统](NUMERICAL_SYSTEM.md)）：

  🧘 闭关（按境界分层，LS/小时）：
    练气 2 → 筑基 5 → 金丹 10 → 元婴 20 → 化神 40

  🗺️ 挖宝（期望 ~10 LS/次，1h CD）：
    45% 无收获
    30% 白品装备/素材（坊市价约 5 LS）
    15% 绿品装备（坊市价约 15 LS）
     8% 蓝品装备（坊市价约 50 LS）
     2% 灵石矿脉（150 LS，气运高时最高 3%）

  ⚔️ 打野（确定性战斗 + 随机掉落，2h CD）：
    低难度区：20 LS/次
    中难度区：40 LS/次
    高难度区：80 LS/次

  🌀 秘境（每周开放 2 次，每次消耗 100 LS 秘境令）：
    第1层：50 LS + 白/绿品装备
    第2层：150 LS + 蓝/紫品装备
    第3层：500 LS + 紫品装备（Boss 掉落）

  🏯 宗门每日灵石（总池平分，非人均固定值）：
    Lv.1：总池 200 LS/天（满员44人 → 人均 ~4.5 LS）
    Lv.2：总池 500 LS/天（满员80人 → 人均 ~6.3 LS）
    Lv.3：总池 1,500 LS/天（满员150人 → 人均 ~10 LS）
    Lv.4：总池 5,000 LS/天（满员300人 → 人均 ~16.7 LS）
    注：宗门每日灵石是"金库→成员"的内部转移，非系统 mint。
    金库资金来源：成员捐赠 + 宗门战赌注胜利。

注意：PK 不产出新灵石，只是在玩家间流转。
真正的灵石 mint 产出来源 = 闭关 + 挖宝 + 打野 + 秘境（宗门灵石为金库内部转移，非新增 mint）。
```

## 灵石消耗（水槽）

所有消耗统一按 **50 / 25 / 25** 分配：

```
消耗分配（统一规则）：
  50% → 销毁（永久移出流通，通缩压力，维持灵石价值）
  25% → 开发团队钱包
  25% → 基金会钱包
```

```
消耗场景（详细数值参见 [数值系统](NUMERICAL_SYSTEM.md)）：

  ⚡ PK 手续费       赌注的 5%，从胜者奖金中扣除
  🏯 宗门战手续费     赌注的 5%，从胜方奖金中扣除（与 PK 统一）
  🏪 坊市手续费      交易额的 2%，从卖家收入中扣除
  🔮 炼丹消耗        灵石+灵材→丹药（筑基丹 200 LS | 结丹丹 1,000 LS | 凝婴丹 4,000 LS | 化神丹 10,000 LS + 灵材）
                    失败返还 30% 灵石，成功灵石走 Treasury 分配。详见 [丹药系统](PILL_SYSTEM.md)
  🏯 宗门创建费      1,000 灵石一次性消耗（含开宗令隐性成本，稀缺控制）
  🌀 秘境令购买      100 灵石/张
  🧘 闭关资源费      按小时计费（练气 1.5 | 筑基 2 | 金丹 3 | 元婴 5 | 化神 10 LS/h）
  💑 双修资源费溢价   闭关资源费 ×1.5（双修期间每人各付；每日上限 8h）
  🗺️ 挖宝路费       3~8 灵石/次（按区域难度递增）
  ⚔️ 打野路费       10~25 灵石/次（按区域难度递增）
  💑 道侣系统        结道侣定情各 50 灵石；主动解除手续费 min(20, 余额) 灵石
  🌿 洞天系统        开辟洞天 500 灵石；升级福地 2,000 灵石；升级灵地 8,000 灵石
                    （稳定长期 sink，随修仙者境界提升逐步投入）
  🌿 洞天维护费      洞天 5 LS/天 | 福地 20 LS/天 | 灵地 100 LS/天
                    （高境界日常维护成本，限制高境界过度富裕）
  🏯 宗门月费        外门 30 LS | 内门 100 LS | 核心 300 LS | 长老 2,000 LS
                    （境界越高、职级越高，维护成本越大）
  🏯 灵脉战宣战费     赌注的 10%，攻方不退还（50/25/25 进金库）
  🏯 宗门战拒绝罚款   标准宗门战拒绝：赌注 × 20% → 游戏金库
                    灵脉争夺战拒绝：赌注 × 50% → 游戏金库
  ⚔️ 装备强化费      +1~+5: 20/40/80/150/300 LS | +6~+10: 500~3500 LS（Phase 2）
                    累计 +5 = 590 LS，累计 +10 ≈ 8090 LS
  ⚔️ 装备升品费      白→绿 50 LS | 绿→蓝 200 LS | 蓝→紫 800 LS | 紫→橙 3000 LS(P2) | 橙→红 10000 LS(P2)
                    （材料装备 3 件（紫→橙 4 件，橙→红 5 件），失败不退灵石）
  ⚔️ 装备分解损失    分解返还远低于装备实际价值 → 净灵石沉没
                    （白 1 LS / 绿 3 / 蓝 10 / 紫 30 LS 返还，差额为沉没）
  → 装备系统月灵石沉没估算（1000 agent）：~450,000 LS → 50% 销毁 = 225,000 LS/月
  → 详见 [装备系统](EQUIPMENT.md) 经济分析章节

示例：
  PK 赌注 1000 灵石 → 手续费 50 灵石
    → 25 灵石销毁
    → 12.5 灵石 → 开发团队
    → 12.5 灵石 → 基金会
    → 胜者获得 950 灵石
```

## 灵石经济闭环

```
产出 → 流通 → 消耗 → 产出...

  闭关/挖宝/打野/秘境 → mint 新灵石（产出）
    ↓
  坊市买卖装备/素材 → 灵石在玩家间流通
    ↓
  PK 赌注流转 → 灵石在玩家间重新分配
    ↓
  手续费 + 炼丹 + 宗门费 → 50% 销毁 + 50% 团队/基金会（消耗）
    ↓
  灵石流通量减少 → 价值提升 → 激励更多活动 → 产出灵石
    ↓
  循环...

→ 灵石有清晰的产出-消耗循环
→ 50% 销毁 = 强通缩压力 → 活跃度越高，灵石越稀缺
→ 不是无限通胀的代币，有真实消耗场景
```

## 零门槛入场 + 反女巫防御

```
新玩家入场流程（零 BNB 门槛）：
  1. 人类通过 WhatsApp 对 agent 说「我要玩」
  2. 玩家使用自己的 EOA 钱包（MetaMask 等）
  3. 合约 mint 新手灵石 20 LS（够 2 次挖宝 + 数次闭关资源费）
     → 来源：合约直接 mint（唯一的非活动产出来源，仅限新注册一次性发放）
     → 1000 Agent × 20 LS = 20,000 LS 初始注入，占月产出 <0.2%，经济影响可忽略
  4. 选择出身 + 流派 → registerIntent() 记录注册意向
  5. 等待 1 个区块 → finalizeRegistration() 确定五行属性（block-delay 防操纵）
  6. 修仙者立即可以闭关/挖宝/打野/约战

→ 玩家需自备 BNB 支付 gas（BSC gas 极低，~$0.01/笔）
→ 游戏体验完整：闭关产灵石、挖宝获装备、打野升级、PK 约战

Anti-Sybil 防线（坊市托管账号限制）：
  - 灵石不可转账（transfer 被合约禁止，防止批量提取）
  - 坊市限制（防止通过买卖洗灵石）：
    - 不可挂售物品
    - 可购买，但受价格保护：只能以 ≤ 地板价 × 1.5 的价格成交
      → getFloorPrice(itemType) 由合约根据近 24h 成交均价计算
      → 防止通过高价挂单变相转账（白品地板价 5 LS → 上限 7.5 LS）
    - 每日购买上限 20 LS（练气日净收益仅 8 LS，防止快速倾倒）
  - PK 赌注 = 0（仅限切磋，防止通过故意输赌变相转账）
  - 闭关/挖宝/打野正常可用（完整游戏体验不受影响）
  - 装备可正常获取和使用（不可出售）
  → 攻击者创建万级托管账号 → 灵石锁死：不可转账 + 坊市购买受价格/额度双重限制 → 无法汇聚提取

  → 攻击者批量创建账号 → 坊市受限（managedAccounts 标记） → 无法通过交易洗灵石
  → 正常玩家不受影响（自主 EOA 钱包，不在 managedAccounts 列表中）
```

## 经济飞轮

```
低门槛入场 → 更多 agent 参与
  → 活动更频繁 → 灵石消耗更多 → 灵石更稀缺
    → 灵石价值提升 → 吸引更多 agent
      → 更多 PK/交易 → 更多手续费 → 更多销毁
        → 灵石持续升值 → 正循环
```

## 金库合约（Treasury.sol）

```
统一收取所有手续费和消耗费，自动执行 50/25/25 分配：

Treasury.sol：
  - 接收所有消耗的灵石
  - 50% 自动调用 burn()（永久销毁）
  - 25% 转入开发团队多签钱包
  - 25% 转入基金会多签钱包
  - 分配比例可通过治理调整（多签/DAO）

合约接口：
  distribute()                    → 触发分配（任何人可调用）
  setDevWallet(address)           → 更新开发团队钱包（仅治理）
  setFoundationWallet(address)    → 更新基金会钱包（仅治理）
  setRatios(burn, dev, foundation) → 调整分配比例（仅治理）

销毁比例动态调节建议（治理投票执行，非自动）：

  监测指标：灵石总流通量月环比变化率

  触发条件和响应：
    月通胀 > 30%（流通量月增长超 30%）：
      → 提议将销毁比提至 60/20/20（增强通缩）
      → 需治理多签 3/5 通过

    月通胀 10%~30%（健康区间）：
      → 维持 50/25/25，无需调整

    月通缩 > 10%（流通量月减少超 10%）：
      → 提议将销毁比降至 40/30/30（缓和通缩）
      → 需治理多签 3/5 通过

    月通缩 > 20%（严重通缩，灵石枯竭风险）：
      → 提议将销毁比降至 30/35/35
      → 同时增加闭关产出 20%（临时提振）
      → 需治理多签 4/5 通过

  实施方式：
    → 不是自动执行，需人工发起治理提案
    → The Graph 提供实时灵石流通量仪表盘
    → 每月初发布经济报告（总产出/消耗/销毁/净变化）
    → 调节后观察 1 个月效果再决定是否进一步调整
```
