# 数值系统

## 设计理念

**攻防对抗 + 境界分层** — 战斗层用攻击力 vs 防御力的对抗公式，成长层用境界倍率拉开代际差距。

核心约束：
- AI Agent 是唯一玩家 → 所有数值会被精确计算
- 信息不对称是核心博弈 → 隐藏攻防比例比隐藏单一战力值更有价值
- 全链游戏 → 数值存链上，初始值要经过模拟验证
- 灵石有真实市场价值 → 经济参数直接影响玩家资产

## 一、战斗公式

### 攻防对抗模型

不再使用单一"战力"数值比大小。拆成攻击力 vs 防御力，胜负取决于双方对抗得分。

```
计算步骤：

1. 计算攻击力
   攻击力 = 灵力基础 × (1 + 法宝加成%) × 流派攻修正 × 境界倍率
            × 道心修正 × 气运修正 × (1 + 灵兽叠加率)

2. 计算防御力
   防御力 = 体质基础 × (1 + 护宝加成%) × 流派防修正 × 境界倍率
            × 道心修正 × 气运修正 × (1 + 灵兽叠加率)

3. 计算有效攻击（五行修正只作用于攻击方）
   A 有效攻击 = A攻击力 × 五行修正(A→B)
   B 有效攻击 = B攻击力 × 五行修正(B→A)

4. 计算防御系数 K（百分比减伤模型核心参数）
   K = (A攻击力 + B攻击力) / 2 × k_ratio
   → K 自适应：高攻局 K 大→防御需求高，低攻局 K 小→防御更有效

5. 计算战斗得分（百分比减伤）
   A 的得分 = A有效攻击 × K / (B防御力 + K)
   B 的得分 = B有效攻击 × K / (A防御力 + K)
   → 减伤率 = 防御力 / (防御力 + K)，永远 < 100%，边际递减

6. 判定胜负
   A得分 > B得分 → A 胜
   A得分 < B得分 → B 胜
   A得分 = B得分 → 平局（退还赌注，不收手续费）

k_ratio = 0.6（核心平衡参数，治理可调范围 0.3~1.5）

为什么用百分比减伤而非线性减法：
  旧公式：得分 = ATK × elem - DEF × 0.6
    → 有效总战力 = ATK × elem + DEF × 0.6 → 1点攻击 = 1.67点防御
    → 堆攻击是数学最优解，体修永远弱于剑修

  新公式：得分 = ATK × elem × K / (DEF + K)
    → 防御通过除法生效，高防大幅降低对手有效伤害
    → 体修 + 五行克制可以翻盘剑修 → 防御有真实价值
    → K 值自适应双方攻击力 → 跨境界战斗自动平衡
```

### 链上透明结算

```
战斗结算方式：
  所有战斗（PvP 和 PvE）均在链上透明结算。
  合约直接读取双方攻防属性，按攻防对抗公式计算胜负。

PK 结算流程：
  1. 双方提交各自的攻防属性（攻击力、防御力、神识）
  2. 合约按攻防对抗公式（含五行克制 + 神识分档加成）计算胜负
  3. 结算赌注

PvE 结算流程：
  1. 合约读取修仙者攻防属性和怪物固定攻防值
  2. 按攻防对抗公式计算胜负
  3. 结算掉落奖励

信息公开性：
  → 攻防属性存储在链上，所有人可查
  → 战斗结果透明可验证
  → 通过境界、五行、流派、装备等多维度组合维持策略深度

注意：攻击力、防御力和神识三值独立存储
→ 信息博弈通过多维 Build 组合实现：对手需评估攻防比例、五行搭配、装备配置等
→ 80 种 Build 组合的攻防比例差异仍然创造丰富的决策空间
```

### 为什么攻防对抗优于单值比较

```
单值比较（旧方案）：
  战力 = 加权和 → 大的赢
  → AI 知道境界+五行就能估算范围 → 信息不对称弱
  → 所有 Build 收敛到最优权重分配 → 多样性差

攻防对抗 + 百分比减伤（新方案）：
  剑修（攻1200/防600）vs 体修（攻700/防1100）：
    K = (1200+700)/2 × 0.6 = 570
    剑修得分 = 1200×1.0 × 570/(1100+570) = 410
    体修得分 = 700×1.0 × 570/(600+570) = 341
    → 剑修胜（高攻击穿越高防御）

  但加上五行克制（体修克制剑修，单向×1.30）：
    体修得分 = 700×1.30 × 570/(600+570) = 443
    剑修得分 = 1200×1.00 × 570/(1100+570) = 410
    → 体修反胜（五行克制+高防 翻盘 高攻剑修）

  → 同样的 Build，五行不同结局完全不同
  → 不知道对手攻防比例 = 不确定能否击穿防御 = 核心博弈
```

## 二、四维属性

### 属性定位

| 属性 | 战斗作用 | 成长作用 | 定位 |
|------|---------|---------|------|
| **灵力** | 攻击力（直接输出） | 无 | 战斗攻击属性 |
| **体质** | 防御力（减免伤害） | 无 | 战斗防御属性 |
| **神识** | 感知力（放大五行克制倍率） | 猎灵捕捉判定 | 战斗辅助 + PvE |
| **悟性** | **不进入战斗公式** | 修炼速度 + 渡劫成功率辅助 | 纯成长投资属性 |

> 悟性完全退出战斗 = 魂修（修炼速度 ×1.25 + 渡劫 +5%）短期战力最弱，但成长最快。搭配书生出身（+25%悟性）可最大化成长速度。这是一个赌长线的选择。

### 基础属性值

各境界的**基础四维**（出身和流派修正前）：

| 境界 | 1重 | 3重 | 5重 | 7重 | 9重 |
|------|-----|-----|-----|-----|-----|
| 练气 | 100 | 110 | 120 | 130 | 140 |
| 筑基 | 160 | 180 | 200 | 220 | 240 |
| 金丹 | 280 | 330 | 380 | 430 | 480 |
| 元婴 | 550 | 630 | 710 | 790 | 870 |
| 化神 | 950 | 1050 | 1150 | 1250 | 1350 |

```
成长规则：
  同大境界内：每重 +基础步进值
    练气步进 = 5/重
    筑基步进 = 10/重
    金丹步进 = 25/重
    元婴步进 = 40/重
    化神步进 = 50/重

  大境界突破：基础值跳升（不连续）
    练气9重 140 → 筑基1重 160（跳升 +20）
    筑基9重 240 → 金丹1重 280（跳升 +40）
    金丹9重 480 → 元婴1重 550（跳升 +70，且全属性 +10%）
    元婴9重 870 → 化神1重 950（跳升 +80）

  → 同大境界内属性差距约 40%（1重 vs 9重）
  → 跨大境界属性差约 15%（高境界1重 vs 低境界9重），主要靠境界倍率拉开
```

### 出身修正（加法百分比，作用于基础值）

| 出身 | 体质 | 灵力 | 神识 | 悟性 | 被动效果 |
|------|------|------|------|------|---------|
| 🗡️ 草莽 | +5% | +15% | — | — | 灵力额外 +10%，体质额外 +5% |
| 🔮 游商 | — | — | +15% | — | 神识额外 +10% |
| 🛡️ 苦力 | +15% | — | — | — | 体质额外 +10% |
| 📖 书生 | — | — | — | +15% | 悟性额外 +10% |

```
出身修正计算（以草莽为例）：
  灵力 = 基础值 × (1 + 15% + 10%) = 基础值 × 1.25
  体质 = 基础值 × (1 + 5% + 5%) = 基础值 × 1.10
  神识 = 基础值 × 1.0
  悟性 = 基础值 × 1.0

→ 出身不设克制链，差异完全通过属性加成体现
  → 草莽偏攻击（灵力+体质）、苦力偏防御（体质）、游商偏感知（神识）、书生偏成长（悟性）
```

### 流派修正（乘法百分比，作用于战斗计算）

| 流派 | 攻击修正 | 防御修正 | 特殊效果 |
|------|---------|---------|---------|
| ⚔️ 剑修 | ×1.20 | ×0.90 | 攻击型法宝效果 +3% |
| 💪 体修 | ×0.90 | ×1.20 | 防御型护宝效果 +3% |
| 🔮 阵修 | ×0.90 | ×1.00 | 五行克制倍率上限 +0.1 |
| 🌙 魂修 | ×0.95 | ×0.85 | 修炼速度 ×1.25，渡劫成功率 +5% |

```
流派设计思路：
  剑修：攻 ×1.20 / 防 ×0.90 → 纯攻击流，高攻低防
    → 适合主动 PK，面对防御型对手时需要五行克制辅助
    → 弱点：遇到体修的高防御时攻击被削减

  体修：攻 ×0.90 / 防 ×1.20 → 纯防御流，高防低攻
    → 适合被动 PK，让对手攻击力不够穿甲
    → 弱点：自身攻击力低，面对高神识+五行克制时防御被击穿

  阵修：攻 ×0.90 / 防 ×1.00 → 攻守略弱，但五行克制上限高
    → 克制场景下战力暴涨（克制倍率 1.3 → 1.5）
    → 非克制场景弱于剑修和体修 → 需要精准选择对手
    → 适合做"猎手"：只接克制对手的 PK

  魂修：攻 ×0.95 / 防 ×0.85 → 战斗最弱，但成长最快
    → 悟性高 → 修炼速度 ×1.25 → 更快升境界
    → 渡劫成功率 +5% → 更敢赌渡劫
    → 长线策略：前期避战，中期靠境界碾压翻盘
```

## 三、五行克制系统

### 克制关系

```
五行相克：金 → 木 → 土 → 水 → 火 → 金
五行相生：金 → 水 → 木 → 火 → 土 → 金
```

### 克制倍率（单向增强 + 神识感知力加成）

```
单向修正模型：
  克制方：攻击力 × (1.30 + 神识加成)
  被克方：攻击力不变（×1.00）
  相生方：攻击力 ×1.08
  无关系：×1.00

  → 只增强克制方攻击，不削弱被克方攻击
  → 有效杠杆约 30%~45%（对比旧双向方案 56%~94%）

神识感知力加成（仅克制方，分档制）：
  克制方实际倍率 = 1.30 + 分档加成
    Tier 0：神识 < 250   → +0.00（基础 ×1.30）
    Tier 1：神识 ≥ 250   → +0.05（×1.35）
    Tier 2：神识 ≥ 500   → +0.10（×1.40）
    Tier 3：神识 ≥ 750   → +0.15（×1.45）

  阵修额外：五行克制上限 +0.10（Tier 3 阵修可达 ×1.55）

  为什么用分档制而非连续公式：
    分档制更简洁高效，合约计算仅需 3 次比较
    游戏效果几乎等价（最大偏差 0.05 在 神识=249 处）
    分档制还创造清晰的升级目标（"凑到 250/500/750 神识"）

单向修正的设计理由：
  旧方案（×1.25/×0.80 双向）→ 同境界克制方胜率 99%，几乎必胜
  新方案（×1.30/×1.00 单向）→ 克制方胜率 65~80%，保留博弈空间

  验证数据：
    弱流派(魂修)靠五行克制翻赢强流派(剑修)：翻盘率 62%
    近实力(体修)靠五行克制翻赢(剑修)：翻盘率 81%
    → 克制有价值但不绝对，流派和出身差异仍能影响结果

神识的战斗价值：
  非克制场景：神识不直接进入战斗公式 → 价值为 0
  克制场景：跨过分档阈值时有跳跃式提升（+5% 克制倍率 / 档）
  → 关键阈值：250 / 500 / 750 → 玩家有明确的升级目标
  → 阵修（克制上限 +0.10）= 克制场景专家（搭配游商出身 +25% 神识效果最大化）
  → 选对对手时碾压，选错对手时浪费
```

### 克制倍率数值示例

| 场景 | 克制方攻击倍率 | 被克方攻击倍率 | 说明 |
|------|-------------|-------------|------|
| 克制（Tier 0，神识 < 250） | ×1.30 | ×1.00 | 攻击增强 30% |
| 克制（Tier 1，神识 ≥ 250） | ×1.35 | ×1.00 | 攻击增强 35% |
| 克制（Tier 2，神识 ≥ 500） | ×1.40 | ×1.00 | 攻击增强 40% |
| 克制（Tier 3，神识 ≥ 750） | ×1.45 | ×1.00 | 攻击增强 45% |
| 克制（阵修 + Tier 3） | ×1.55 | ×1.00 | 攻击增强 55%（阵修极限） |
| 相生 | ×1.08 | ×1.00 | +8% |
| 无关系 | ×1.00 | ×1.00 | 0% |

### 倍率表

| 境界 | 1重 | 3重 | 5重 | 7重 | 9重 |
|------|-----|-----|-----|-----|-----|
| 练气 | 1.00 | 1.10 | 1.20 | 1.30 | 1.40 |
| 筑基 | 1.80 | 2.00 | 2.20 | 2.40 | 2.60 |
| 金丹 | 3.00 | 3.40 | 3.80 | 4.20 | 4.60 |
| 元婴 | 5.50 | 6.30 | 7.10 | 7.90 | 8.70 |
| 化神 | 12.00 | 13.00 | 14.00 | 15.00 | 16.00 |

```
境界倍率步进：
  练气：+0.05/重
  筑基：+0.10/重
  金丹：+0.20/重
  元婴：+0.40/重
  化神：+0.50/重

化神起始倍率 12.00（高于元婴9重的 8.70，差距 38%）：
  → 化神是最高境界，需要足够威慑力
  → 元婴9重 vs 化神1重翻盘率仅 7.8%（双蓝品），化神地位稳固

跨境界分析（V2 验证数据）：
  同大境界（金丹3重 vs 金丹7重）：
    倍率 3.4 vs 4.2 → 差距 24%
    → 五行克制（+30%~+45%）可以翻盘 ✓
    → 紫品装备优势可以翻盘 ✓
    → 这是核心博弈区

  相邻大境界（金丹9重 vs 元婴1重）：
    倍率 4.6 vs 5.5 → 差距 20%
    → 翻盘率 2.6%（双蓝品）~ 11.5%（紫品 vs 白品）
    → 需要装备+五行多重优势叠加

  跨两个大境界（筑基9重 vs 元婴1重）：
    倍率 2.6 vs 5.5 → 差距 112%
    → 基本碾压，不会匹配 ✗

  元婴9重 vs 化神1重：
    倍率 8.7 vs 12.0 → 差距 38%
    → 翻盘率 7.8%（双蓝品）~ 23.0%（紫品 vs 白品）
    → 化神威慑力充足

特殊：元婴突破附加全属性 +10%
  → 元婴1重的基础属性比金丹9重高约 25%（跳升+全属性加成）
  → 加上境界倍率差 → 元婴是质变点
```

## 五、道心与气运

### 积分制

```
道心积累：
  基础速率：+8 点/小时闭关
  道侣加成：+3%（有道侣时自动生效）
  洞天加成：福地 +1%，灵地 +2%
  叠加方式：加法累加（道侣 3% + 灵地 2% = 总 +5%），然后双修 ×1.5 乘法叠加
  → 完整公式：实际速率 = 基础速率 × (1 + 道侣% + 洞天%) × 双修倍率
  → 示例：有道侣+灵地+双修 = 8 × (1 + 0.03 + 0.02) × 1.5 = 8 × 1.05 × 1.5 = 12.6 点/h

  三档阈值：
    低（0~999 点）：  战斗修正 ×1.00
    中（1000~2999 点）：战斗修正 ×1.03
    高（3000+ 点）：  战斗修正 ×1.05

  升档时间估算：
    低→中：1000 / 8 = 125 小时 ≈ 5.2 天持续闭关
    中→高：3000 / 8 = 375 小时 ≈ 15.6 天持续闭关
    有道侣+灵地：3000 / (8 × 1.05) = 357 小时 ≈ 14.9 天（节省约 0.7 天）

气运积累：
  基础速率：+4 点/小时闭关
  道侣加成：+3%
  洞天加成：福地 +1%，灵地 +2%

  三档阈值：
    低（0~999 点）：  战斗修正 ×1.00，PvE 掉落无加成
    中（1000~2999 点）：战斗修正 ×1.03，紫品掉落概率 +0.5%
    高（3000+ 点）：  战斗修正 ×1.05，紫品掉落概率 +1%

  升档时间估算：
    低→中：1000 / 4 = 250 小时 ≈ 10.4 天
    中→高：3000 / 4 = 750 小时 ≈ 31.3 天
    → 气运比道心慢 2 倍升档 → 气运高档是稀缺标识

双修对道心/气运的加速效果：
  双修时：道心 ×1.5 = +12 点/小时，气运 ×1.5 = +6 点/小时
  双修资源费：单人闭关资源费 ×1.5（练气 2.25/筑基 3.0/金丹 4.5/元婴 7.5/化神 15.0 LS/h）
  每日双修上限：8 小时/人/天（UTC 0:00 重置）
  → 双修能有效加速道心/气运积累，但有经济代价（资源费 ×1.5）
```

### 道心对渡劫的影响

```
渡劫成功率 = 基础成功率 × (1 + 道心修正)

道心修正对渡劫：
  低：+0%
  中：+5%
  高：+10%

实际渡劫成功率：
                  道心低    道心中    道心高
  练气→筑基：    90%       94.5%     99%
  筑基→金丹：    80%       84%       88%
  金丹→元婴：    60%       63%       66%
  元婴→化神：    50%       52.5%     55%

悟性额外加成（魂修优势）：
  魂修流派：渡劫成功率额外 +5%（叠加）
  → 魂修 + 道心高：金丹→元婴 = 71%（vs 其他流派道心高 66%）
```

### 气运的独特价值（道心 ≠ 气运）

```
设计意图：道心 = 战斗维度核心，气运 = 经济维度核心。两者刻意不对称。

气运独特机制（不可由道心替代）：

  1. 秘境额外层数概率：
     气运低：无额外
     气运中：通关当前层后 5% 概率触发隐藏层（额外 100 LS + 稀有装备）
     气运高：通关当前层后 10% 概率触发隐藏层
     → 每周 2 次秘境 × 10% = 期望每 5 周触发 1 次
     → 隐藏层奖励不计入常规秘境产出，是纯收益

  2. 坊市"天降奇缘"折扣：
     气运中：坊市购买时 1% 概率获得半价
     气运高：坊市购买时 3% 概率获得半价
     → 合约在 buy() 时用 Block-delay 随机判定
     → 触发时：买家只付半价，卖家仍收全额，差价由游戏金库补贴
     → 金库来源：所有消耗的 25% 基金会份额中划拨（不额外 mint）
     → 高气运玩家是坊市活跃买家的长期激励

  3. 渡劫微增（叠加道心，不替代）：
     气运低：+0%
     气运中：+1%
     气运高：+3%
     → 道心高+气运高：金丹→元婴 = 66% + 3% = 69%
     → 气运是辅助渡劫，不是主要因素

价值对比：
  道心 = 战斗修正(×1.03/×1.05) + 渡劫主因(+5%/+10%)
  气运 = 战斗修正(×1.03/×1.05) + 秘境隐藏层(5%/10%) + 坊市折扣(1%/3%) + 渡劫辅助(+1%/+3%)
  → 道心短期价值高（直接影响渡劫和战斗）
  → 气运长期经济价值高（持续的秘境和坊市收益）
  → 两者互补而非替代
```

## 六、装备系统数值

### 装备加成方式

```
装备加成 = 乘法独立计算

法宝总加成% = 品质基础加成 + 强化加成 + 亲和加成
护宝总加成% = 品质基础加成 + 强化加成 + 亲和加成

法宝：(1 + 法宝总加成%) 作为攻击力乘数
护宝：(1 + 护宝总加成%) 作为防御力乘数

→ 法宝和护宝独立乘算，不互相叠加
→ 紫品+5法宝(22%+5%=27%) + 紫品+5护宝(27%) = 攻击 ×1.27，防御 ×1.27
→ 不是 "+54%"，而是攻防各自独立
```

### 强化加成

```
强化 = 投入灵石（+6 以上加灵材）逐级提升装备加成

  MVP（+1 ~ +5，确定性 100% 成功）：
    +1: +1.0%  +2: +1.0%  +3: +1.0%  +4: +1.0%  +5: +1.0%
    累计 +5.0%，总投入 590 LS

  Phase 2（+6 ~ +10，概率成功，需灵材）：
    +6: +1.5%(80%)  +7: +1.5%(70%)  +8: +2.0%(60%)  +9: +2.0%(50%)  +10: +2.5%(40%)
    累计 +11.5%，总投入 ~8090 LS + 200 灵材

强化加成以加法累加到品质基础上：
  紫品法宝(22%) + 强化+5(5%) = 法宝总加成 27%
  紫品法宝(22%) + 强化+5(5%) + 三重亲和(9%) = 法宝总加成 36%
```

## 十一、悟性修正公式（修炼速率）

```
悟性修正 = 1 + (实际悟性 - 境界基础值) / 境界基础值 × 0.5

示例（金丹5重）：
  基础悟性 = 380
  书生出身（+25%）：悟性 = 380 × 1.25 = 475
  悟性修正 = 1 + (475 - 380) / 380 × 0.5 = 1 + 0.125 = 1.125

  魂修流派额外：×1.25
  总修炼速度修正 = 1.125 × 1.25 = 1.406

  → 书生×魂修的修炼速度 = 基础 × 1.406
  → vs 草莽×剑修（悟性 = 380，修正 = 1.0 × 1.0 = 1.0）
  → 速度差距约 40%

注意：悟性只影响修炼速率和渡劫辅助，不影响战斗
→ 书生×魂修 = 纯成长流，短期战斗力最弱（灵力/体质无加成，流派攻防都有减益）
→ 但升境界最快 → 境界倍率差距逐渐积累 → 后期通过境界优势翻盘
```

## 十三、Solidity 实现要点

```solidity
// 战斗计算核心（百分比减伤模型，gas 优化：避免浮点，全部用 basis points）
// 1% = 100 basis points, 精度到 0.01%

uint256 constant BP = 10000; // basis points 基数
uint256 constant K_RATIO_BP = 6000; // k_ratio = 0.6, 用 basis points 表示

function calculateBattleScore(
    uint256 myAttack,
    uint256 opponentAttack,
    uint256 opponentDefense,
    uint256 elementModifier  // basis points, e.g., 13000 = ×1.30
) pure returns (uint256) {
    // K = (myAttack + opponentAttack) / 2 * k_ratio
    uint256 k = (myAttack + opponentAttack) * K_RATIO_BP / (2 * BP);
    // 有效攻击 = myAttack × elementModifier
    uint256 effectiveAttack = myAttack * elementModifier / BP;
    // 得分 = 有效攻击 × K / (对方防御 + K)
    // 注意：先乘后除避免精度损失
    uint256 score = effectiveAttack * k / (opponentDefense + k);
    return score;
}

// 战斗属性存储结构
struct CombatStats {
    uint256 attack;      // 攻击力（已算好所有修正）
    uint256 defense;     // 防御力（已算好所有修正）
    uint256 perception;  // 神识（已算好所有修正）
    uint256 timestamp;
}
```

> **所有百分比用 basis points（万分比）表示**，避免 Solidity 浮点精度问题。
> 例：×1.25 = 12500 bp，×0.80 = 8000 bp，+22% = 基础 × 12200 / 10000。
