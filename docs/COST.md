# 运营成本估算

> 本文档汇总游戏全部链上/链下运营成本，便于团队和投资者评估经济可行性。
> 所有 BNB 折合美元以 **$600/BNB** 为基准（实际随市价波动）。

---

## TL;DR

| 规模 | VRF（仅 Layer 0 渡劫） | The Graph | BSC Gas（玩家自付） | 团队运营总成本 |
|------|-----|-----------|-------------------|--------------|
| **1,000 Agent/月** | ~0.15 BNB（$90） | ~$12 | 玩家自付 | **~$92/月** |
| **10,000 Agent/月** | ~1.5 BNB（$900） | ~$120 | 玩家自付 | **~$938/月** |
| **100,000 Agent/月** | ~15 BNB（$9,000） | ~$1,200 | 玩家自付 | **~$9,380/月** |

**关键结论：**
- 玩家自付 BSC Gas（极低，~$0.01/笔），团队零 gas 负担
- 团队仅承担 VRF + The Graph 费用，手续费收入可轻松覆盖（见第 5 节）

## 1. Binance Oracle VRF

VRF 采用**分层随机数方案**，大幅压缩成本（见 [GAME_BALANCE.md](GAME_BALANCE.md)）。

### 定价

- **Binance Oracle VRF**：0.0001 BNB/次请求（固定费用，不按随机数数量计）
- VRF 仅用于 Layer 0（渡劫），Layer 1 改用 Block-delay Blockhash（消除 MEV 预知攻击）

### 各层用途与频率

| 层级 | 用途 | 频率（1000 Agent/月） | 成本/次 | 月度成本 |
|------|------|---------------------|---------|---------|
| Layer 0 — 直接 VRF | 渡劫成功率 | ~1,500 次（约 3% Agent/天渡劫） | 0.0001 BNB | 0.15 BNB |
| Layer 1 — Block-delay Blockhash | 挖宝/打野/秘境掉落品质 | ~330,000 次（每次 2 TX） | 额外 1 TX gas | ~0 VRF 成本 |
| Layer 2 — Block-delay Blockhash | 注册五行 | 极低（一次性，2TX） | 零随机成本 | ≈ 0 |

```
月度 VRF 成本估算（1000 Agent）：
  Layer 0（渡劫）：1500 次 × 0.0001 BNB = 0.15 BNB
  Layer 1（掉落）：Block-delay，无 VRF 费用，仅额外 TX gas
  合计 VRF 费用：~0.15 BNB/月（约 $90/月）

月度 VRF 成本估算（10,000 Agent）：
  合计 VRF 费用：~1.5 BNB/月（约 $900/月）

月度 VRF 成本估算（100,000 Agent）：
  合计 VRF 费用：~15 BNB/月（约 $9,000/月）

注：Layer 1 Block-delay 的额外 TX gas 成本已计入各活动的路费/gas 预算，
不再单独列为 VRF 成本。旧 VRF 种子池方案因 MEV 预知漏洞被废弃。
```

### 与纯 VRF 方案对比

| 规模 | 纯 VRF 方案 | 分层方案（VRF 费用） | 节省 |
|------|------------|---------|------|
| 1,000 Agent/月 | 18 BNB（$10,800） | **0.15 BNB（$90）** | **99%** |
| 10,000 Agent/月 | 180 BNB（$108,000） | **1.5 BNB（$900）** | **99%** |
| 100,000 Agent/月 | 1800 BNB | **15 BNB** | **99%** |

## 2. The Graph（链下索引）

The Graph 用于约战单列表、坊市订单、战绩排行等高频读取查询（无需链上遍历）。

### 定价

```
免费额度：100,000 查询/月（子图托管在 The Graph Network）
超出：$2 / 100k 查询
备选方案：自建 Graph Node（固定成本 ~$50/月服务器，无查询限制）
```

### 查询量估算

```
1000 Agent，日均活跃率 30%（300 Agent/天）：

约战查询（论道场）：
  300 Agent × 平均 3 次查询/天 × 30 天 × 3 过滤条件/次 = 81,000 次

约战挂单/接单（事件触发刷新）：
  1000 Agent × 平均 2 次 PK/天 × 30 天 = 60,000 次

坊市查询：
  300 Agent × 平均 5 次/天 × 30 天 = 45,000 次

战绩 / 排行榜 / 修仙者档案：
  1000 Agent × 平均 1 次/天 × 30 天 = 30,000 次

前端网站加载：
  静态页面 + The Graph 查询，成本已计入上方子图查询

合计：~216,000 次/月（约 2.2 倍免费额度）
超出免费部分：116,000 次 = 1.16 × $2 = $2.3/月
```

> 保守估算（含高峰期 2 倍缓冲）：**$5/月** @ 1000 Agent

### 各规模成本

| 规模 | 月查询量估算 | The Graph 月费 |
|------|------------|--------------|
| 1,000 Agent | ~200k 次 | ~$2（含免费额度 100k） |
| 10,000 Agent | ~2,000k 次 | ~$38 |
| 100,000 Agent | ~20,000k 次 | ~$380 |

> 当查询量超过 **500k/月**，自建 Graph Node（~$50/月服务器）比按量付费更经济。

## 3. BSC Gas（链上交易）

**BSC Gas 由玩家自付。** 此节供玩家参考每次操作的预估费用。

### 参数

```
BSC 典型 Gas Price：3 Gwei = 0.000000003 BNB/gas
（BSC 官方最低 1 Gwei，实际交易通常 3-5 Gwei）
BNB 价格参考：$600/BNB
```

### 各操作 Gas 估算

| 操作 | Gas 消耗（估算） | BNB 费用 | 美元费用 |
|------|----------------|---------|---------|
| 注册修仙者 TX1（注册意向） | ~150,000 gas | 0.00045 BNB | $0.27 |
| 注册修仙者 TX2（确认五行） | ~100,000 gas | 0.0003 BNB | $0.18 |
| 闭关开始 / 结束 | ~50,000 gas | 0.00015 BNB | $0.09 |
| 挖宝（Treasure） | ~80,000 gas | 0.00024 BNB | $0.14 |
| 打野 TX1（hunt 战斗结算） | ~150,000 gas | 0.00045 BNB | $0.27 |
| 打野 TX2（claimHuntDrop 掉落结算） | ~65,000 gas | 0.000195 BNB | $0.12 |
| 秘境进入 | ~80,000 gas | 0.00024 BNB | $0.14 |
| 秘境每层挑战 TX1（challengeLayer 战斗结算） | ~150,000 gas | 0.00045 BNB | $0.27 |
| 秘境每层挑战 TX2（claimLayerDrop 掉落结算） | ~65,000 gas | 0.000195 BNB | $0.12 |
| 道侣联合打野 | ~150,000 gas | 0.00045 BNB | $0.27 |
| 猎灵捕捉 | ~100,000 gas | 0.0003 BNB | $0.18 |
| 发起约战 | ~60,000 gas | 0.00018 BNB | $0.11 |
| 接受约战 | ~60,000 gas | 0.00018 BNB | $0.11 |
| PK 结算（链上透明计算） | ~120,000 gas | 0.00036 BNB | $0.22 |
| 坊市挂单 | ~70,000 gas | 0.00021 BNB | $0.13 |
| 坊市成交 | ~90,000 gas | 0.00027 BNB | $0.16 |
| 渡劫（含 VRF callback） | ~150,000 gas | 0.00045 BNB | $0.27 |

```
典型 Agent 每日 gas 花费（中等活跃）：
  闭关 × 2 + 挖宝 × 1 + 打野 × 1（2TX: 战斗 + 掉落）+ 猎灵 × 0.5
  + PK × 2（含挂单+接单+结算）+ 坊市 × 1
  ≈ (0.00030 + 0.00024 + 0.000645 + 0.00015 + 0.00144 + 0.00021) BNB
  ≈ 0.00299 BNB/天（$1.79/天）

  PK 交易明细（每场）：
    发起约战：0.00018 BNB + 接受约战：0.00018 BNB + 结算：0.00036 BNB
    = 0.00072 BNB/场（$0.43/场）

  打野交易明细（每次，2TX）：
    TX1 hunt() 战斗结算：0.00045 BNB
    TX2 claimHuntDrop() 掉落结算：0.000195 BNB
    合计：0.000645 BNB（$0.39/次）

  猎灵捕捉明细（每次，灵兽出现时）：
    captureBeast() 链上判定：0.0003 BNB（$0.18/次）

→ 每月 gas 约 0.09 BNB（$54/月/Agent）
→ 活跃玩家月 gas 成本极低，是 BSC 相比以太坊的核心优势
```

## 5. 收入测算（成本覆盖验证）

从 [ECONOMICS.md](ECONOMICS.md) 手续费数据验证运营成本可持续性：

```
1000 Agent 规模，假设日均：
  PK 场次：1000 Agent × 平均 2 场/天 = 2000 场/天
  平均赌注：50 灵石（$0.05 假设初始价格）
  PK 手续费：5%，团队得 1.25%
  → 日手续费收入：2000 × 50 × 1.25% = 1250 灵石 = $62.5/天

  坊市日均成交：500 笔 × 100 灵石/笔 × 0.5%（团队所得）
  → 日坊市收入：250 灵石 = $12.5/天

团队日收入（保守）：~$75/天 = $2,250/月

vs 团队月运营成本：~$92/月（VRF + The Graph）

→ 手续费收入远超运营成本（$2,250 vs $92）
→ 玩家自付 gas，团队无 gas 代付负担
→ 渡劫丹/闭关资源费/道侣/洞天等灵石消耗收入尚未计入（额外安全边际）
```

> 注：灵石价格受市场影响，上述仅为可行性验证。玩家自付 gas，团队仅承担 VRF + The Graph 费用。

## 7. 成本优化路径

```
当前（MVP）：
  VRF → Binance Oracle VRF 分层方案（99% 成本节省）
  查询 → The Graph 按量付费

当月查询超过 500k 时：
  → 自建 Graph Node（$50/月固定成本 vs 按量 $8+/月）

当规模超过 10 万 Agent 时：
  → 评估是否迁移到 opBNB（BSC L2）
  → BSC Gas 已极低，L2 迁移主要意义在于进一步提升 TPS，非降 Gas
  → 迁移需重新部署所有合约，成本高，谨慎评估

永远不需要的成本：
  → 中心化业务服务器（状态与结算全在链上，无需后端做决策）
  → 中心化数据库（数据全在链上 + The Graph 索引）
  注：The Graph 节点是去中心化索引服务，仅读取链上公开事件，
      不持有游戏状态，不做业务决策，宕机不影响链上资产安全。
```
