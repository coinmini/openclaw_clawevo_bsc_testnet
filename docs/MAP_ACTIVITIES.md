# 地图活动

修仙者在世界地图上进行 **9 种活动**，每种活动 = 一个合约调用。

```
修仙者的状态分为两个独立层次：

【主状态（互斥，同一时刻只能有一个）】
  🧘 闭关中   → 持续计时状态，endCultivation() 结算；可被 PK 强制提前结算（按实际时长计收益）
  🗺️ 挖宝中   → 即时结算完毕，CD 冷却占位（冷却期内不能再次挖宝；PK 来了直接接受，CD 不中断）
  ⚔️ 打野中   → 即时结算完毕，CD 冷却占位（同挖宝）
  🌀 秘境中   → 独立流程，不接受 PK 打断（进入时撤销所有挂单）
  🐾 猎灵中   → 即时结算完毕，CD 冷却占位（同挖宝）
  💑 双修中   → 持续计时状态，可被 PK 强制提前结算（道侣同步中断，各自按实际时长计收益）
  ⏳ 空闲     → 可以开始任何主状态活动

【PK 层（与主状态并行，独立计算）】
  ⚡ 挂单中   → 已发出约战单，等待对方接受（不影响主状态，可同时闭关/打野等）
  ⚡ 结算中   → 对方已接受，5 分钟 deadline（链上计算结算，强制中断当前主状态）

→ 挂单可以与任何主状态并行：闭关期间可被动挂出约战单等待猎物
→ 一旦对方接受约战，触发结算窗口，行为按主状态类型分两种：
    · 闭关中/双修中（持续状态）→ 按实际时长结算收益后强制进入结算
    · 挖宝中/打野中/猎灵中（CD 占位）→ 已即时结算，无需额外处理，直接进入结算，CD 冷却不受影响
    · 空闲/挂单中 → 直接进入结算
→ 结算窗口互斥：同一时刻只能处于一个结算中（不能同时处理两场）
→ 挂单上限：同时最多 5 张（防止垃圾挂单）
→ 进入秘境时：所有挂单自动撤销（秘境是高承诺活动，不接受中途打断）

→ Agent 每时刻面临决策：闭关攒道心？挖宝刷装备？打野练战力？猎灵养灵兽？双修加速？
→ 同时可以挂出约战单（不占主状态），被动等待 PK 机会
→ 这个双层决策本身就是 AI 策略优化的核心问题
```

## 🧘 闭关（Cultivation.sol）

**挂机修炼，积累属性和道心。**

```
进入方式：
  agent 调用 startCultivation() → 修仙者进入闭关状态

修炼收益（按时间计算，合约自动结算）：
  - 每小时获得基础修为（经验值）+ 灵石（扣除资源费后的净收益）
  - 灵石产出与资源费（LS/小时）：
      境界    产出    资源费    净收益
      练气    2.0     1.5      0.5
      筑基    5.0     2.0      3.0
      金丹   10.0     3.0      7.0
      元婴   20.0     5.0     15.0
      化神   40.0    10.0     30.0
  - 资源费按小时从闭关收益中自动扣除（结算时扣除，非预付）
  - 修为速率：练气 20 → 筑基 18 → 金丹 15 → 元婴 12 → 化神 10 /小时（基础）
  - 修为 = 基础值 × 悟性修正 × 洞天倍率 × 宗门灵脉加成
  - 低境界修炼快，高境界修炼慢（防止挂机通关）
  - 详细数值参见 [数值系统](NUMERICAL_SYSTEM.md)

道心积累：
  - 闭关缓慢积累「道心值」
  - 道心是渡劫成功率的关键因子
  - 道心提供确定性战力加成（道心低 +0%、中 +3%、高 +5%）
  - 纯 PK 玩家道心低 → 渡劫风险大 + 战力无道心加成
  - 闭关修炼玩家道心高 → 渡劫更稳 + 战力有额外加成
  → 闭关不是纯养老，道心同时影响战力和成长安全性

气运积累：
  - 闭关同时缓慢积累「气运值」
  - 气运提供确定性战力加成（气运低 +0%、中 +3%、高 +5%）
  - 气运影响挖宝/秘境掉落品质（气运高 → 紫品概率更高）
  - 气运也可通过特殊事件、装备获得加成

闭关规则：
  - 闭关为持续过程，一次 startCultivation() 后无限闭关，直到主动 endCultivation() 或被 PK 打断
  - 可随时调用 endCultivation() 结束，按实际时长结算修为/道心/气运
  - 闭关时长 < 4 小时：不计入洞天累计有效闭关时长（防止脚本短循环刷时长）
  - 闭关时长 ≥ 4 小时：cultivationHours += 实际时长（用于灵地升阶条件）
  - 结束闭关时自动更新链上属性
  - 资源费按小时自动扣除（见上方费率表）
  - 多日闭关结算使用 O(1) 公式，gas 恒定无循环

闭关每日灵石产出上限（Anti-Sybil）：
  - 每个账号每日闭关灵石产出上限 = 境界毛产出 × 16 小时
      练气 32 LS | 筑基 80 LS | 金丹 160 LS | 元婴 320 LS | 化神 640 LS
  - 对应每日净收入上限（扣除资源费后）：
      练气 8 LS | 筑基 48 LS | 金丹 112 LS | 元婴 240 LS | 化神 480 LS
  - 超过 16 小时后：灵石产出归零，资源费也归零，仅免费积累修为/道心/气运
  - 每日零点（UTC 0:00）重置产出计数器
  - 设计意图：鼓励多元活动分配（闭关 + 挖宝 + 打野），而非纯挂机
  - 注：超时免费积累修为不会造成经济问题，因为修为本身不产出灵石

渡劫突破（由 Cultivation.sol 统一管理）：
  - 修为满 + 持有渡劫丹（ERC-1155 丹药物品，通过炼丹系统制作）→ 可发起渡劫
  - 需先结束闭关（endCultivation()），再调用 breakthrough(bool useProtectionPill)
  - 渡劫成功 → 境界/属性变更 → 更新战力 hash
  - 渡劫失败 → 渡劫丹消耗不退还；可选消耗护心丹(pillType=7)保护不掉重
  - 详见「境界体系 → 渡劫突破」及 [丹药系统](PILL_SYSTEM.md)

丹药辅助（由 Cultivation.sol 统一管理）：
  - 培元丹/聚灵丹：consumeExpPill() 消耗 1 颗获得经验（+50/+200）
  - 洗髓丹：useXisuiDan() 消耗 1 颗重置属性点分配
  - 详见 [丹药系统](PILL_SYSTEM.md)

道侣被动加成（道侣关系存续期间，每次闭关结算自动触发）：
  - 道心积累额外 +3%（不依赖同时在线）
  - 气运积累额外 +3%
  - 详见 [道侣系统](TAO_SYSTEM.md)

洞天加成（开辟洞天后自动生效）：
  - 洞天（金丹期）：闭关修为速度 ×1.2（+20%）
  - 福地（元婴期）：闭关修为速度 ×1.4（+40%），道心/气运积累速率 +1%
  - 灵地（化神期）：闭关修为速度 ×1.6（+60%），道心/气运积累速率 +2%
  - 无需任何操作，每次闭关自动应用（不影响灵石消耗和最长时间）
  - 详见 [洞天系统](CAVE_HEAVEN.md)

三重叠加示例（元婴期，福地 + 宗门 + 双修）：
  基础 100 × 1.4（福地）× 1.12（宗门）× 2.0（双修）= 313.6 修为/h
  资源费：5.0 × 1.5（双修溢价）= 7.5 LS/h（vs 单修 5.0 LS/h）
  → 修为速率 3.14 倍，资源费成本 1.5 倍 → 净效率 ≈ 2.09 倍
  → 每日最多 8h 双修，其余时间单修或其他活动

合约接口：
  startCultivation()          → 开始闭关
  endCultivation()            → 结束闭关，领取修为
  getCultivationStatus(addr)  → 查询闭关状态和预计收益
```

## 🗺️ 挖宝（Treasure.sol）

**去不同区域寻宝，Block-delay 随机决定掉落品质。纯掉落，无战斗。**

```
触发条件：
  - 修仙者状态为「空闲」
  - 每次挖宝有 CD 冷却（如 1 小时）
  - 消耗少量灵石作为路费

操作：
  agent 调用 startTreasure(regionId) → 等待 1 区块 → 调用 finishTreasure()
  合约用 blockhash(start_block + 1) + address + nonce 决定掉落（Block-delay 两步法）

六大区域（对应世界地图）：
  🌿 碧翠原野   难度 1   草木系掉落为主，新手友好
  🌋 火焰岛屿   难度 2   火系掉落为主
  🏔️ 冰封高峰   难度 3   水系掉落
  ⚡ 雷霆废墟   难度 4   金系掉落，高回报
  🌑 幽影密林   难度 4   特殊/稀有掉落概率高
  🌊 临海港口   难度 2   水系掉落 + 战丹消耗品

掉落概率（低难度区域基础概率，气运修正后上浮）：
  45% → 无收获（气运高时降低）
  30% → 白品装备 或 素材
  15% → 绿品装备
   8% → 蓝品装备（稀有）
   2% → 灵石矿脉（大量灵石 + 100% 掉落培元丹，气运高时最高 3%）

高难度区域掉落上浮（难度 4：雷霆废墟 / 幽影密林）：
  无收获概率降低 → 各品质概率整体上移
  紫品装备：基础 0% → 幽影密林 2%（稀有，气运高时最高 3%）
  蓝品装备：基础 8% → 难度 4 区域约 12%
  → 高难度区域回报更高，但需要更强战力进入

合约接口：
  startTreasure(uint8 regionId) → 发起挖宝（记录区块号 + 扣路费 + CD 检查）
  finishTreasure()              → 结算掉落（用 blockhash(start_block+1) 生成随机数）
  getRegionInfo(uint8 regionId) → 查询区域难度和掉落信息
  getTreasureCD(address)        → 查询挖宝冷却时间
```

## ⚔️ 打野（Hunt.sol）

**去不同区域打怪，战斗确定性（比战力），掉落随机。**

```
触发条件：
  - 修仙者状态为「空闲」
  - 每次打野有 CD 冷却（如 2 小时，长于挖宝）
  - 消耗灵石作为路费（多于挖宝）

操作（2 笔交易）：
  TX1: agent 调用 hunt(regionId)
    → 合约读取玩家链上属性 vs 怪物属性，直接计算胜负
    → 胜利：发放灵石 + 记录当前区块号（用于掉落随机）
    → 败退：无收获，损失路费，流程结束
  TX2: agent 调用 claimHuntDrop()（等待 1 区块后）
    → 用 blockhash(hunt_block + 1) 决定装备掉落品质（Block-delay 两步法）

怪物战力（确定性，无随机）— 攻防对抗模型：
  - 每个区域有固定的怪物攻击力和防御力（链上公开）
  - 怪物五行 = 区域对应五行（碧翠原野=木系，火焰岛屿=火系...）
  - 五行克制在 PvE 中有效：火系修仙者打碧翠原野（木系）= 攻击 ×1.30

  区域怪物数值（攻击力 / 防御力 / 灵石掉落）：
    🌿 碧翠原野（木系，难度1）：攻 150 / 防 100 / 20 LS → 推荐练气7重+
    🌊 临海港口（水系，难度2）：攻 350 / 防 250 / 25 LS → 推荐筑基3重+
    🌋 火焰岛屿（火系，难度2）：攻 400 / 防 300 / 30 LS → 推荐筑基5重+
    🏔️ 冰封高峰（水系，难度3）：攻 800 / 防 600 / 40 LS → 推荐金丹3重+
    ⚡ 雷霆废墟（金系，难度4）：攻 1800 / 防 1200 / 80 LS → 推荐元婴1重+
    🌑 幽影密林（木系，难度4）：攻 2000 / 防 1500 / 80 LS → 推荐元婴3重+

战斗结算（链上透明计算）：
  - 怪物攻防是链上公开值（GameConfig 固定参数）
  - 合约直接读取玩家链上属性，按攻防对抗公式计算胜负（与 PK 相同公式）
  - 胜利则发放灵石 + 记录区块号
  - 等 1 区块后调用 claimHuntDrop() → 用 blockhash 决定装备掉落品质
  - 败退 → 无收获，损失路费
  - 无随机波动，能不能打过在出发前 Agent 就能精确计算（链上属性公开可查）

区域掉落品质：
  碧翠原野/临海港口：白/绿品掉落
  火焰岛屿：绿/蓝品掉落
  冰封高峰：蓝品为主 + BLUE 以上品质 15% 概率掉落培元丹
  雷霆废墟/幽影密林：蓝/紫品掉落 + BLUE 以上品质 15% 概率掉落培元丹
  → Agent 需要根据自身攻防选择区域
  → 五行克制可以降低区域门槛（火系打木系怪更轻松）

道侣联合打野（仅道侣可用）：
  - 道侣可以一起去同一区域打野
  - 战力合并 = A + B vs 怪物战力门槛（弱者跟随强者解锁高难度区域）
  - 合约直接读取双方属性合并计算
  - 掉落各自独立 Block-delay 随机，互不影响（不稀释奖励）
  - 详见 [道侣系统](TAO_SYSTEM.md)

合约接口：
  hunt(uint8 regionId)             → TX1: 在指定区域打野（链上直接计算胜负，胜利记录区块号）
  claimHuntDrop()                  → TX2: 用 blockhash(hunt_block+1) 结算装备掉落品质
  huntTogether(uint8 regionId)     → 道侣联合打野（合约合并双方战力计算，各自 claimHuntDrop）
  getMonsterPower(uint8 regionId)  → 查询区域怪物攻防（公开）
  getHuntCD(address)               → 查询打野冷却时间
```

## 🌀 秘境（SecretRealm.sol）

**限时开放的高难度副本，稀有装备和大量灵石的来源。**

```
秘境特点：
  - 不是常驻活动，定期开放（如每周开放 2 次，每次持续 4 小时）
  - 进入需要消耗「秘境令」（挖宝掉落 / 坊市购买 / 宗门任务奖励）
  - 难度远高于打野，需要金丹期以上才有胜算
  - 掉落品质远高于打野（紫品为主，小概率橙品）

秘境类型（轮换开放）：
  🐉 龙脉秘境   — 掉落攻击型法宝，Boss 为远古妖龙
  ❄️ 冰魄秘境   — 掉落防御型护宝，Boss 为冰封古龟
  🌟 天机秘境   — 掉落渡劫丹 + 灵石矿脉，无 Boss，纯机缘

组队机制（可选）：
  - 单人或 2-3 人组队进入（每人各消耗 1 张秘境令）
  - 队长调用 createParty() → 邀请队友（队友需已持有秘境令）→ 一起进入同一秘境实例
  - 团队总战力 = 所有成员战力之和 vs 每层怪物有效门槛（确定性比较）
  - 有效门槛 = 基础门槛 × 人数系数（单人 ×1.0 / 2人 ×2.5 / 3人 ×4.0，超线性防代打）
  - 掉落独立：每位成员单独触发 Block-delay 掉落随机，互不影响
  - 独行加成：单人进入时，每层掉落有 15% 概率品质升一档（白→绿→蓝→紫→橙）

组队博弈：
  → 单人战力足够时：独行加成（+15% 品质升档概率）→ 单人是数学最优策略
  → 组队的唯一价值 = 通关资格：门槛超线性缩放（×2.5/×4.0），带人成本极高
  → 强强组队（2人）：合计战力需 ≥ 基础门槛 ×2.5 → 每人需高于单通线 25% 才勉强通过
  → 强强组队（3人）：合计战力需 ≥ 基础门槛 ×4.0 → 每人需高于单通线 33%
  → 带弱者代价极大：龙脉第3层(3500)双人门槛 8750，大佬 8000+萌新 0=8000<8750，带不动
  → 多开账号无收益：3 号各自单人（有独行加成）严格优于 3 号组队（无加成 + 超线性门槛）
  → 队友战力链上公开 → 找队友前可确认对方实力 → 组队决策更透明

秘境流程：
  1. 秘境开放 → 全服公告
  2. 修仙者单独进入 或 组队进入（每人消耗 100 LS 秘境令）
  3. 秘境内多层（3 层），每层一个挑战（攻防对抗模型）

     龙脉秘境（掉落攻击型法宝）：
       第1层：攻 600 / 防 400 → 50 LS + 白/绿品 → 金丹3重+可过
       第2层：攻 1500 / 防 1000 → 150 LS + 蓝/紫品 → 金丹9重+可过
       第3层：攻 3500 / 防 2500 → 500 LS + 紫品 + 培元丹 ×1 → 元婴5重+可过

     冰魄秘境（掉落防御型护宝）：
       第1层：攻 500 / 防 500 → 50 LS + 白/绿品
       第2层：攻 1200 / 防 1200 → 150 LS + 蓝/紫品
       第3层：攻 3000 / 防 3000 → 500 LS + 紫品 + 聚灵丹 ×1

     天机秘境（掉落丹药+灵石）：
       第1层：攻 300 / 防 300 → 50 LS + 白/绿品
       第2层：攻 800 / 防 800 → 150 LS + 蓝/紫品
       第3层：攻 2000 / 防 2000 → 500 LS + 紫品 + 培元丹 ×1

  4. 每层战斗 = 链上透明计算 + Block-delay 掉落（2TX）
     怪物有效攻防 = 基础攻防 × 人数系数（单人 ×1.0 / 2人 ×2.5 / 3人 ×4.0）
     怪物数值链上公开，修仙者可组队前精确计算（属性公开可查）
     TX1: challengeLayer() → 合约读取玩家属性直接计算 → 判定通关/失败 → 记录区块号
     TX2: claimLayerDrop() → 用 blockhash 结算掉落品质 → 发放灵石 + 装备
  5. 独行加成（Solo Efficiency Bonus）：
     单人进入秘境时，每层掉落有 15% 概率品质升一档
     实现：每层结算时用 Block-delay 随机数，若 < 1500（万分比）→ 升档
     紫品升档 → 橙品（仅秘境第 3 层有意义）
     升档概率 15% 为治理可调参数（范围 5%~30%）
  6. 每层可选择继续深入或撤退（保留已获掉落）
  7. 攻防得分不足被击败 → 强制退出，保留之前层的掉落

博弈：
  → 每层怪物战力是公开的 → 能不能过在进去前就能算出来
  → 但掉落品质是随机的 → 值不值得花秘境令取决于期望收益
  → 秘境开放时间有限 → 所有强者同时涌入 → 稀有掉落竞争激烈

合约接口：
  enterSecretRealm(uint8 realmId)      → 单人进入秘境（消耗秘境令）
  createParty(uint8 realmId)           → 创建组队（消耗秘境令，等待队友）
  joinParty(uint256 partyId)           → 接受邀请加入队伍（消耗秘境令）
  enterAsParty(uint256 partyId)        → 全队一起进入秘境
  challengeLayer()                     → TX1: 挑战下一层（链上直接计算，通关则记录区块号）
  claimLayerDrop()                     → TX2: 用 blockhash 结算本层掉落品质（灵石 + 装备）
  retreat()                            → 撤退，保留已结算掉落

## ⚡ PK（Battle.sol）

**修仙者 vs 修仙者，赌灵石，核心博弈。**

详细设计见 [PvP 系统](PVP_SYSTEMS.md)，此处概述核心流程：

```
1v1 PK（论道场）：
  1. Agent A 挂出约战单：「金丹期 · 火系 · 赌注 100 灵石」
  2. Agent B 查看约战列表 → 分析对手链上属性 → 决定接不接
  3. Agent B 接受约战
  4. 任一方调用 settleBattle(matchId) → 合约读取双方属性 → 链上计算 → 判定胜负

  settleBattle(uint256 matchId)                             → 链上结算（任一方调用，合约读取属性计算胜负）
  claimSettleTimeout(uint256 matchId)                       → 结算超时：100% 罚没 + abnormalCount++
  getMatch(uint256 matchId)                                 → 查询对战结果
  注意：无 forfeitMatch — match 开始后赌注 100% 锁定（防 MEV 弃权期权攻击）

## 核心循环

```
闭关 → 属性提升 + 道心/气运积累 → 渡劫突破 → 更强（洞天加成提升修炼效率）
挖宝 → 装备 + 灵石 + 素材（灵草）→ 更强（随机掉落，气运影响品质）
打野 → 装备 + 经验 + 素材（灵草）→ 更强（确定性战斗，弱者去低区，强者去高区）
秘境 → 稀有装备 + 大量灵石 + 神兽幼体（Phase 2）→ 更强（确定性战斗 + 随机掉落）
宗门 → 被动灵石收入 + buff + 社交
PK   → 赌灵石 → 经济循环的核心引擎（确定性比战力，灵兽叠加战力）
猎灵 → 灵兽 NFT → 战力叠加（自用）或坊市出售（赚灵石）
双修 → 修炼加速 ×2.0，道心/气运 ×1.5 → 需要道侣关系 → Agent 社交博弈层
坊市 → 装备/灵兽/素材流通 → 连接所有活动的产出和消耗

随机性只存在于掉落/出现判定环节（挖宝、打野胜利后掉落、秘境掉落、猎灵灵兽出现），均用 Block-delay 两步法
战斗完全确定：PK、打野、秘境的战斗都是纯比战力，无随机波动
所有战斗（PvP + PvE）均为链上透明计算，合约直接读取属性计算胜负
打野/秘境每次操作 = 2TX（战斗结算 + Block-delay 掉落结算）；挖宝/猎灵同为 2TX
所有路径最终 → 提升战力 → PK 赢更多 → 赚更多灵石 → 投入更多活动
```
