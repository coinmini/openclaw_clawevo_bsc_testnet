# 灵兽系统

灵兽是修仙者在天地间捕捉到的灵化野物，经过驯化后可以上阵参与战斗。
灵兽是 **ERC-721 NFT**，每只独一无二，可在坊市自由交易。

> 灵兽 = 第三个战力维度。法宝强攻，护宝固防，灵兽全面叠加总战力。
> 战力公式：`总战力 = 修仙者基础战力 × ... × (1 + 灵兽叠加率)`

---

## 灵兽定位

```
任何流派的修仙者都可以携带一只灵兽参战。
灵兽信息链上可查。

对手可以查到你是否携带灵兽及其星级，
但灵兽 × 装备 × 流派 × 五行的多维组合
仍然创造丰富的战力差异和策略空间。

例：
  金丹期 · 火系 · 剑修 · 紫品法宝 + 3星异兽
  vs
  金丹期 · 火系 · 剑修 · 紫品法宝（无灵兽）
  → 前者实际战力高出 20%，对手无法从公开信息判断
```

## 灵兽星级体系

| 星级 | 名称 | 战力叠加 | 获取方式 |
|------|------|---------|---------|
| ⭐ 1 星 | 凡兽 | +5% | 猎灵（所有区域，常见） |
| ⭐⭐ 2 星 | 灵兽 | +12% | 猎灵（中高难度区域） |
| ⭐⭐⭐ 3 星 | 异兽 | +20% | 猎灵（幽影/雷霆，需缚灵索）— Phase 2 |
| ⭐⭐⭐⭐ 4 星 | 神兽幼体 | +30% | 秘境专属掉落 — Phase 2 |
| ⭐⭐⭐⭐⭐ 5 星 | 上古神兽 | +40% | Phase 3+ |

```
战力叠加计算示例：
  基础战力 1000 × ... × (1 + 0.20) = 1200
  → 3 星异兽为修仙者提供 +200 战力（全乘）
  → 无灵兽时，公式中 (1 + 0) = 1，对战力无影响
```

### 物种抽取规则

```
捕获时的物种决定方式：
  区域 + 星级 → 从该区域同星级物种池中随机抽取（Block-delay 随机）
  例：在碧翠原野猎灵，出现 1 星 → 50% 翠叶鼠 / 50% 藤蔓蛇
  例：在碧翠原野猎灵，出现 2 星 → 100% 碧角麋鹿

物种影响：
  物种仅影响外观和收集图鉴，不影响战力数值。
  战力叠加率由星级 + Block-delay 随机浮动决定（见上方星级体系）。

Phase 2+ 扩展：
  3 星异兽（Phase 2）和 4-5 星神兽（Phase 3+）将在后续版本扩展图鉴。
```

## 灵兽 NFT 属性

```
每只灵兽 NFT 包含以下链上属性：

  基础属性：
    - 物种名称（如「赤焰狐」「玄冰蟾」）
    - 星级（1-5 星，铸造时确定，不可降级）
    - 五行属性（与所在区域五行一致，铸造时确定）
    - 战力叠加率（同星级有小幅浮动：±2%，铸造时 Block-delay 随机）

  状态属性（Phase 2 引入）：
    - 当前等级（1-9 级，可通过灵草素材升级）
    - 已喂食灵草总量（链上记录成长历史）

→ 同为 2 星灵兽，五行属性不同 + 叠加率浮动 → 每只价值不同
→ 坊市估值时需 Agent 综合评估（五行匹配主人 + 叠加率浮动）
```

### 五行亲和（微加成）

```
灵兽五行与主人五行相同时：
  相同五行 → 亲和加成 +1%（在叠加率基础上额外 +1%）
  五行相生 → 亲和加成 +0.5%
  无关联   → 无额外加成

→ 选择与自己五行匹配的灵兽，收益略高
→ 但稀有灵兽（高星）即使五行不匹配也远胜低星同五行
```

## 猎灵活动

详细规则见 [地图活动](MAP_ACTIVITIES.md#-猎灵beasthuntsol)。

```
核心流程：
  1. 空闲 → 选区域 → 调用 startHuntBeast(regionId) → 等待 1 区块 → finishHuntBeast()
  2. Block-delay 随机决定是否出现灵兽及星级
  3. 确定性判定：合约读取修仙者神识值，
     判定"神识 ≥ 灵兽抵抗力 × 星级系数"
  4. 成功 → 铸造灵兽 NFT；失败 → 灵兽逃脱，获少量灵草

出现概率（MVP，Block-delay 随机决定，气运高时 2 星上浮略优）：
  70% 无灵兽出没 / 22% 出现 1 星 / 8% 出现 2 星
  3 星异兽：Phase 2 开放，MVP 权重为 0

捕捉判定（确定性，链上透明计算）：
  判定公式：神识值 ≥ 灵兽抵抗力 × 星级系数
  结算方式：合约直接读取神识值并比较（Gas ~100,000）
  灵兽抵抗力：按区域固定（碧翠原野 80 / 临海港口 120 / 火焰岛屿 150 / 冰封高峰 250 / 雷霆废墟 400 / 幽影密林 500）
  星级系数：1 星 ×1.0 / 2 星 ×3.0 / 3 星 ×5.0（Phase 2）

  达标率估算（同境界修仙者中满足条件的比例）：
    1 星：多数金丹期修仙者达标（~90%+ 的 build 组合）
    2 星：需较高神识（游商出身优势明显，~70% 的金丹期 build 达标）
    3 星（Phase 2）：需元婴期 + 缚灵索 + 高神识（~50% 的元婴期 build 达标）

  注：百分比描述的是"同境界多少种 build 组合满足阈值"，不是每次捕捉的随机概率
```

## 灵兽培养（Phase 2）

```
灵兽升级体系：
  灵兽等级：1-9 级（对应境界小境界）
  升级材料：灵草素材（挖宝/打野副产品）
  升级上限：灵兽等级 ≤ 主人境界等级（金丹期 3 重 → 灵兽最高 3 级）

实际叠加率 = 基础叠加率 × (1 + 0.05 × 灵兽等级)

示例：
  2 星 9 级：12% × (1 + 0.45) = 17.4%
  3 星 1 级：20% × (1 + 0.05) = 21.0%
  → 高星低级 vs 低星高级 存在交叉，培养路线产生博弈

灵草素材消耗（每级递增）：
  1→2 级：50 灵草
  2→3 级：80 灵草
  3→4 级：120 灵草
  ... 等比递增
  → 灵草素材成为新的紧俏资源，驱动挖宝/打野活动频率
```

### 灵兽繁殖（Phase 2）

```
两只相同五行的灵兽可以繁殖：
  消耗：双方各消耗 1 次繁殖次数（每只灵兽最多繁殖 3 次）
  结果：生成新的灵兽 NFT，星级由 Block-delay 随机决定：
    双亲均 1 星 → 后代 1 星 95% / 2 星 5%
    双亲均 2 星 → 后代 2 星 90% / 3 星 10%
    双亲 1+2 星 → 后代 1 星 50% / 2 星 45% / 3 星 5%

→ 繁殖是获取高星灵兽的额外途径，也是灵兽 NFT 的长期价值支撑
→ 繁殖需消耗灵石（繁殖费 → 金库），驱动经济循环
```

## 合约接口

```solidity
// Beast.sol — 灵兽 NFT（ERC-721）
contract Beast {
    struct BeastInfo {
        uint8 star;        // 1-5 星
        uint8 element;     // 五行（同区域五行）
        uint8 level;       // 等级（Phase 2，MVP 默认 1）
        uint16 powerRate;  // 战力叠加率（单位 0.01%，如 1200 = 12%）
        string species;    // 物种名称
    }

    function getBeast(uint256 tokenId) external view returns (BeastInfo);
    function getOwnerBeast(address owner) external view returns (uint256[] tokenIds);
}

// BeastHunt.sol — 猎灵活动
contract BeastHunt {
    function startHuntBeast(uint8 regionId) external;   // Step 1: 发起猎灵（Block-delay）
    function finishHuntBeast() external;                 // Step 1b: block-delay 判定灵兽出现及星级
    function captureBeast(                               // Step 2: 链上捕捉判定
        uint256 requestId
    ) external;
    function getHuntCD(address hunter) external view returns (uint256 cdEnd);
    function getRegionBeastTypes(uint8 regionId) external view returns (uint8[] stars, uint16[] probabilities);

    event BeastAppeared(address indexed hunter, uint8 star, uint8 element);
    event BeastCaptured(address indexed hunter, uint256 indexed tokenId, uint8 star);
    event BeastEscaped(address indexed hunter, uint8 star);
}
```

## 灵兽交易规则

```
灵兽为 ERC-721 NFT，通过坊市（Marketplace 合约）交易。

交易基础规则：
  坊市手续费：成交额 × 2%（与装备交易一致）
    → 分配：50% 销毁 / 25% 团队 / 25% 基金会
  挂单/撤单：免费（仅消耗 gas）
  支持拍卖模式和一口价模式

交易冷却（防刷）：
  新捕获灵兽：铸造后 24 小时交易冷却期
    → 防止猎灵机器人高频捕捉-挂牌套利
  交易后新主人：无额外冷却（二手交易自由流通）
  繁殖后代（Phase 2）：铸造后 48 小时交易冷却

绑定规则（MVP）：
  1 星凡兽：自由交易
  2 星灵兽：自由交易
  3 星异兽（Phase 2）：首次绑定 7 天后方可交易
    → 鼓励自己猎灵而非坊市直购
  4 星+ 神兽（Phase 2+）：不可交易（灵魂绑定 Soulbound）
    → 最高星级灵兽只能自己获得，不可用灵石购买力碾压

设计意图：
  → 1-2 星自由交易：低价灵兽流通性好，新手可快速获取
  → 3 星限时绑定：高价值灵兽有获取门槛，抑制纯买家策略
  → 4 星+ Soulbound：终局灵兽不可交易 = 真实实力的标志
  → 富人可以买 2 星灵兽跳过猎灵，但 3 星+ 必须亲自参与
```

## 经济影响

```
新增灵石消耗场景：
  猎灵路费          → 少量灵石（每次猎灵）
  繁殖费（Phase 2） → 中量灵石 → 金库

新增素材消耗场景（Phase 2）：
  灵草素材用于培养   → 驱动挖宝/打野产出更被重视
  缚灵索消耗品       → 捕 3 星必需，来自坊市/秘境掉落

新增 NFT 市场流动性：
  灵兽 NFT 在坊市交易 → 与装备 NFT 同一市场
  坊市手续费 2%      → 灵石消耗水槽
  高星灵兽价格随稀有度走高 → 正向激励猎灵活动
  繁殖次数有限（每只 3 次） → 控制供给，维持稀缺性
```
