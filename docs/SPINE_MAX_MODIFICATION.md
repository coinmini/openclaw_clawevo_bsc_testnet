# Spine 角色最大限度视觉改造

## Context

对 `need_change_color/1/` Spine 角色做最大限度修改：颜色以及一切可改的视觉变换。
角色是修仙女性，105 个 PNG 部件，深色头发 + 奶白/金色衣甲 + 桃色肤色 + 白色特效。

## 修改项目清单（9 步流水线）

### A. 颜色类（在全分辨率上操作）

| 步骤 | 变换 | 适用部件 | 效果 |
|------|------|---------|------|
| 1 | **选择性换色** — HSV 范围匹配，将衣甲金/奶色(H:15-55) → 深红(H:0)，深色头发 → 银白 | 身体69 + 特殊6 | 极高 |
| 2 | **气流色相偏移** — +180° | 气流16帧 | 高 |
| 3 | **冷色温偏移** — R-15, B+15 | 身体 + 特殊 | 中 |
| 4 | **对比度增强** — factor 1.3 | 身体 + 特殊 | 中 |
| 5 | **饱和度增强** — factor 1.4 | 身体 + 特殊 | 中 |
| 6 | **色调分离** — posterize 5 bits | 身体 + 特殊（跳过 <10px 部件）| 高 |
| 7 | **锐化** — UnsharpMask(1, 80) | 身体 + 特殊 | 低 |

### B. Alpha/特效类

| 步骤 | 变换 | 适用部件 | 效果 |
|------|------|---------|------|
| 8 | **Alpha 增强** — ×1.5 | 气流16帧 | 中 |
| 9 | **辉光/泛光** — 提取亮像素 → 高斯模糊 → 叠加 | 剑光14帧 | 高 |

### 部件分类规则

```
qi_effects    = qiliu_1_*.png           → 步骤 2, 8
sword_effects = sword_light1_*.png      → 步骤 9
shadow        = diying.png              → 无变换
body+special  = 其余所有                 → 步骤 1, 3-7
```

## 执行顺序的原因

颜色操作在全分辨率上执行，保留原始细节。

## 技术要点

- **RGBA 安全**：所有 ImageOps 函数需分离 alpha 通道，仅处理 RGB
- **选择性换色**：用 numpy 向量化 HSV 转换，按色相/饱和度/明度范围匹配替换
- **Atlas 重建**：解析 `.atlas` 坐标 + rotation 标记，逐区域裁剪 → 变换 → 粘贴回去
  - 非旋转：`crop(x, y, x+w, y+h)`
  - 旋转90°：`crop(x, y, x+h, y+w)`
- **小部件保护**：<10px 部件跳过 posterize

## 输出结构

```
need_change_color/1_modified/
    images/     ← 105 个处理后的单独 PNG
    1.png       ← 重建的 atlas（2033×840）
    1.atlas     ← 原样复制
    1.skel      ← 原样复制
```

## 实现

脚本：`scripts/modify_spine_max.py`
- 依赖：PIL/Pillow, numpy
- 每个变换函数接收 Image 返回新 Image（不可变）
- 按部件分类自动应用对应流水线

## 验证

1. 输出目录有 105 个 PNG，尺寸与原始一致
2. Atlas 尺寸 2033×840
3. 目视检查：颜色明显改变（金→红，发→银白）
